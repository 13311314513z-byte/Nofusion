# 文本导入方案修改：单次上限提升至 500 万字 + 分片提取

> 编制日期：2026-06-06  
> 适用范围：NoFusion / InkOS 1.4.1 当前工作区  
> 关联文档：`书籍模块强化.md`（书籍资料包导入方案）  
> 目标：将文本导入单次最高硬上限从当前 50 万字提升至 **500 万字**，并实现分片提取机制以支持超大文件。

---

## 一、现状与差距分析

### 1.1 当前限制全景

| 层级 | 位置 | 当前值 | 目标值 |
|------|------|--------|--------|
| Core 常量 | `document-reader.ts` → `MAX_CHARS` | **500,000** | **5,000,000** |
| Core 截断逻辑 | `processExtractedText()` | `.slice(0, maxChars)` 直接截断 | 分片处理 + 截断标记 |
| Core 提取函数 | `extractFromJsonl()` / `extractFromJson()` | 全量加载到内存 | 分批迭代器 |
| API import-url | `server.ts` → `maxChars` 校验 | `Math.min(..., 500_000)` | `Math.min(..., 5_000_000)` |
| API body 读取 | `readStyleImportBody()` | **2,000,000** 字节 | **20,000,000** 字节 |
| API 文件大小 | content-length 检查 | **5 MB** | **50 MB** |
| API extract-text | `/style/extract-text` | 不接收 `maxChars` | 接收 `maxChars` 参数 |
| Frontend | `StyleManager.tsx` | 无分片感知 | 分片加载 UI |

### 1.2 10 倍提升的核心挑战

| 挑战 | 说明 |
|------|------|
| **内存压力** | 500 万字原始文本约 10MB（UTF-8），JSONL 解析后 AST 可能膨胀 2-3x |
| **单次处理耗时** | 超大 JSONL 逐行解析 + `isValidTextValue` 验证可能阻塞事件循环数十秒 |
| **API 响应体大小** | 单次返回 500 万字 JSON 响应体约 10MB，可能触碰 Hono/Bun 默认限制 |
| **前端渲染** | React 一次渲染 500 万字文本会导致编辑器卡死 |
| **Hash 计算** | `computeTextHash` 对 500 万字做 SHA-256 约需 50-100ms（尚可接受） |

## 二、总体方案

### 2.1 核心策略

```
┌─────────────────────────────────────────────────────────────────┐
│                        调用方（Frontend/CLI）                     │
└──────────────────────────┬──────────────────────────────────────┘
                           │ POST /style/extract-text
                           │ { text, fileType, maxChars, chunk? }
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│  API 层（server.ts）                                              │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ 1. 校验 maxChars ≤ 5_000_000                                │ │
│  │ 2. 调用 core 分片提取                                        │ │
│  │ 3. 返回当前分片 + 分片元信息                                  │ │
│  └─────────────────────────────────────────────────────────────┘ │
└──────────────────────────┬───────────────────────────────────────┘
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│  Core 层（document-reader.ts）                                    │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ 1. MAX_CHARS = 5_000_000                                   │ │
│  │ 2. extractFromJsonlChunked() — 分批迭代器                   │ │
│  │ 3. extractFromJsonChunked()  — Array/Object 分片           │ │
│  │ 4. extractDocumentChunked()  — 统一入口                     │ │
│  │ 5. ExtractedDocument 增加分片元信息字段                      │ │
│  └─────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────┘
```

### 2.2 架构原则

1. **向后兼容**：现有 500K 以下的调用不受影响，`extractDocument` / `extractDocumentFromText` 签名不变
2. **按需分片**：仅当 `processed.length > maxChars` 或原始文件超过阈值时才触发分片
3. **无状态分片**：每个分片请求独立计算，服务端不维护分片会话状态
4. **流式友好**：分片提取器使用 Generator / AsyncGenerator，便于未来接入流式响应

#### 1.3 新增分片提取函数

##### JSONL 分片迭代器

```typescript
/**
 * 将 JSONL 文本按行分批提取，每批达到 maxCharsPerChunk 时 yield。
 * 调用方可逐批消费，避免单次处理超大文件。
 */
export function* extractFromJsonlChunked(
  text: string,
  maxCharsPerChunk: number = MAX_CHARS_PER_CHUNK,
): Generator<{ text: string; chunkIndex: number; isLast: boolean }> {
  const lines = text.split("\n");
  const currentChunk: string[] = [];
  let currentLength = 0;
  let chunkIndex = 0;

  for (const line of lines) {
    const trimmed = line.trim();
    if (!trimmed) continue;

    let extracted: string[] = [];
    try {
      const obj = JSON.parse(trimmed);
      // ... 现有提取逻辑 ...
      extracted = extractTextFromJsonValue(obj);
    } catch {
      if (isValidTextValue(trimmed, "fallback")) {
        extracted = [trimmed];
      }
    }

    for (const part of extracted) {
      if (currentLength + part.length > maxCharsPerChunk && currentChunk.length > 0) {
        yield {
          text: currentChunk.join("\n\n"),
          chunkIndex: chunkIndex++,
          isLast: false,
        };
        currentChunk.length = 0;
        currentLength = 0;
      }
      currentChunk.push(part);
      currentLength += part.length;
    }
  }

  if (currentChunk.length > 0) {
    yield {
      text: currentChunk.join("\n\n"),
      chunkIndex,
      isLast: true,
    };
  }
}
```

#### 1.4 修改 `processExtractedText` 签名

当前 `processExtractedText` 直接 `.slice(0, maxChars)` 截断。改为返回截断信息：

```typescript
function processExtractedText(
  text: string,
  fileType: DocumentFileType,
  maxChars: number,
): { text: string; truncated: boolean; originalLength: number } {
  let processed = text;
  // ... 按 fileType 提取 ...
  
  const originalLength = processed.length;
  if (processed.length > maxChars) {
    processed = processed.slice(0, maxChars);
    return { text: processed, truncated: true, originalLength };
  }
  return { text: processed, truncated: false, originalLength };
}
```

#### 1.5 修改 `extractDocument` / `extractDocumentFromText`

适配新的 `processExtractedText` 返回结构，填充 `ExtractedDocument` 新增字段：

```typescript
export function extractDocumentFromText(
  text: string,
  sourceName: string,
  fileType: DocumentFileType = "txt",
  options?: { maxChars?: number },
): ExtractedDocument {
  const maxChars = options?.maxChars ?? MAX_CHARS;
  const { text: cleaned, truncated, originalLength } = processExtractedText(text, fileType, maxChars);
  const warnings = validateExtractedText(cleaned, fileType);
  
  return {
    sourceName,
    fileType,
    text: cleaned,
    charCount: cleaned.length,
    textHash: computeTextHash(cleaned),
    warnings,
    truncated,
    originalLength: truncated ? originalLength : undefined,
    totalChunks: truncated ? estimateTotalChunks(originalLength, maxChars) : 1,
    chunkIndex: 0,
  };
}
```

### Phase 2：API 层 — 分片端点 + 上限更新

**涉及文件：** `packages/studio/src/api/server.ts`

#### 2.1 更新 `/api/v1/style/extract-text`

```typescript
app.post("/api/v1/style/extract-text", async (c) => {
  const { text, sourceName, fileType, maxChars, chunk } = 
    await c.req.json<{
      text: string;
      sourceName: string;
      fileType?: DocumentFileType;
      maxChars?: number;   // 新增
      chunk?: number;      // 新增：请求指定分片索引
    }>();

  if (!text?.trim()) return c.json({ error: "text is required" }, 400);
  
  // 校验并钳位 maxChars
  const effectiveMaxChars = typeof maxChars === "number" && Number.isFinite(maxChars)
    ? Math.min(Math.max(maxChars, 1000), MAX_CHARS)
    : MAX_CHARS;

  // 如果请求了指定分片，使用分片提取器
  if (typeof chunk === "number" && chunk >= 0) {
    const gen = extractDocumentChunked(text, sourceName ?? "sample", fileType ?? "txt", {
      maxChars: effectiveMaxChars,
    });
    let index = 0;
    for (const doc of gen) {
      if (index === chunk) return c.json(doc);
      index++;
    }
    return c.json({ error: `chunk ${chunk} out of range (total ${index})` }, 404);
  }

  // 默认：返回全量（或第一个分片）
  try {
    const extracted = extractDocumentFromText(text, sourceName ?? "sample", fileType ?? "txt", {
      maxChars: effectiveMaxChars,
    });
    return c.json(extracted);
  } catch (e) {
    return c.json({ error: String(e) }, 500);
  }
});
```

#### 3.3 分片加载流程

```typescript
const handleExtractText = async () => {
  // 1. 请求第一片
  const doc = await fetchJson<ExtractedDoc>("/style/extract-text", {
    method: "POST",
    body: JSON.stringify({ text: fileText, sourceName, fileType, maxChars: 5_000_000 }),
  });
  
  setChunks([doc]);
  setExtractedDoc(doc);  // 当前显示的文本

  // 2. 如果有多片，后台加载剩余分片
  if (doc.truncated && doc.totalChunks > 1) {
    loadRemainingChunks(doc.totalChunks);
  }
};

const loadRemainingChunks = async (totalChunks: number) => {
  setLoadingChunk(true);
  const loaded = [chunks[0]];
  for (let i = 1; i < totalChunks; i++) {
    try {
      const chunk = await fetchJson<ExtractedDoc>("/style/extract-text", {
        method: "POST",
        body: JSON.stringify({ text: fileText, sourceName, fileType, maxChars: 5_000_000, chunk: i }),
      });
      loaded.push(chunk);
      setChunks([...loaded]);
    } catch (e) {
      setChunkError(`第 ${i + 1}/${totalChunks} 片加载失败`);
      break;
    }
  }
  setLoadingChunk(false);
  
  // 全部加载完成后，合并文本
  if (loaded.length === totalChunks) {
    const allText = loaded.map(c => c.text).join("\n\n");
    setExtractedDoc({
      ...loaded[0],
      text: allText,
      charCount: allText.length,
      truncated: false,
      totalChunks,
      chunkIndex: 0,
    });
  }
};
```

### Phase 4：边界处理与降级策略

#### 4.1 内存安全

| 场景 | 处理方式 |
|------|----------|
| 原始文件 500 万字纯文本 | 约 10MB，单次加载可接受 |
| 原始文件 > 100MB | API 层拒绝（content-length 检查） |
| JSONL 行数 > 100 万行 | 分片提取器按行分批，每批 10,000 行 |
| 单行 JSON > 10MB | 流式读入，拒绝超长单行 |

#### 4.2 错误处理

- 分片加载失败：不影响已加载的分片，允许用户重试失败分片
- 分片超时：`AbortSignal.timeout(60000)` 覆盖下载 + 提取全流程
- 内存溢出：Node.js 本身抛出 `RangeError`，API 层捕获并返回 413

#### 4.3 降级路径

```
请求 500 万字导入
  ├─ 文件 ≤ 50MB ──→ 单次全量提取（内存安全）
  ├─ 文件 > 50MB  ──→ 拒绝（提示用户拆分为多个文件）
  ├─ 提取结果 ≤ 500 万字 ──→ 单分片返回
  └─ 提取结果 > 500 万字 ──→ 多分片返回（每片 50 万字）
```

## 七、与书籍模块强化的关系

`书籍模块强化.md` 提出了 `FoundationSourceBundle` 资料包层，本方案为该资料包的**前置能力**——让超大文本资料能够完整导入并传递给 Architect 进行架构生成。两者结合后的调用链：

```
超大文件（≤500 万字）
  → 分片提取（本方案）
  → 分片拼接为完整资料包
  → FoundationSourceBundle（书籍模块强化）
  → Architect 生成/修订
  → 写入 story/outline、roles、pending_hooks
```

> **本方案不依赖书籍模块强化**，可独立上线。书籍模块强化可复用本方案的分片提取能力。
