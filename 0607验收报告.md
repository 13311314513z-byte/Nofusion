### 2.2 `style-adjuster.ts`（280 行）

**符合方案要求：**
- ✅ 纯函数，接收 `FullStyleDiagnostics` 作为输入
- ✅ 确定性补丁规则：
  - `action-expression` 重复 → 删除第二次出现
  - 段落开头连续过渡词 → 删除过渡词
- ✅ `suggestion.id` 使用 `stableId(ruleVersion, category, position, evidence)`
- ✅ severity 排序：`critical > warning > info`，同级别按 confidence 降序
- ✅ `maxSuggestions` 默认 20

**潜在问题：**

```typescript
// 第 93-102 行：buildIntentPatch
const second = finding.examples[1];
const snippet = text.slice(second.start, second.end);
```

- ⚠️ 直接取 `examples[1]` 做删除，若两例相距很远（>200 字符），删除可能导致语义断裂。建议增加距离检查。

```typescript
// 第 110-122 行：buildTransitionPatch
const lines = text.split("\n");
for (let i = 1; i < lines.length; i++) {
```

- ⚠️ `text.split("\n")` 假设段落以换行分隔，但中文小说常以 `。` 结尾无换行。该补丁在实际文本中可能很少命中。

**严重缺失：**
- ❌ **无测试文件**。`style-adjuster.test.ts` 不存在。
