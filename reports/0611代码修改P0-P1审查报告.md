## P0 — 必须立即修复

### P0-1 `POST /api/v1/style/rhetoric/rewrite` 500 崩溃
- **文件**：`packages/studio/src/api/server.ts`
- **行号**：5532–5558
- **现象**：`TypeError: Cannot read properties of undefined (reading 'map')`，返回 500。
- **根因**：
  1. `raw.findings ?? (...)` 仅对 `null/undefined` 生效；如果调用方传了 `findings: null` 或非数组值，代码会原样保留，随后 `as any[]` 强转并调用 `.map()` 崩溃。
  2. `detectDuplicateRhetoric(raw.text, "zh").findings` 可能返回 `undefined`，导致 `findings` 为 `undefined`。
- **修复**：
  ```ts
  let findings: unknown[] | undefined;
  if (Array.isArray(raw.findings)) {
    findings = raw.findings;
  } else if (Array.isArray(raw.categories) && raw.categories.length) {
    findings = raw.categories.map((cat) => ({
      category: cat,
      label: cat,
      count: 0,
      perThousandChars: 0,
      severity: "low" as const,
      examples: [] as Array<{ text: string }>,
    }));
  } else {
    findings = detectDuplicateRhetoric(raw.text, "zh").findings;
  }
  if (!Array.isArray(findings)) {
    return c.json({ error: "findings must be an array" }, 400);
  }
  const prompt = buildDedupePrompt(raw.text, findings as any[], (raw.mode ?? "replace") as any);
  ```

### P0-2 `style-library` 作者蒸馏路径遍历
- **文件**：`packages/core/src/style-library/distillation-store.ts`
- **行号**：29–59（所有 path helper）
- **现象**：恶意 `authorId`（如 `../../../etc`）可导致项目目录外的读写。
- **根因**：`authorId` 原样拼接到 `join(root, "style-library", "authors", authorId, ...)`。
- **修复**：

  ```ts
  function assertSafeAuthorId(id: string): string {

  ```
  在每个导出的 CRUD 函数入口调用 `assertSafeAuthorId(authorId)`。

## P1 — 安全与运行时缺陷

### 1. SSRF：`POST /api/v1/style/authors/fetch`
- **文件**：`packages/studio/src/api/server.ts`
- **行号**：5640–5654
- **风险**：服务端按任意 `url` 发起 `fetchUrl`，可访问内网 / localhost / `file://`。
- **修复**：添加 URL 白名单与主机黑名单校验（仅允许 `http/https`，禁止回环及内网地址）。

### 2. 路径遍历：`POST /api/v1/style/authors/samples/write`
- **文件**：`packages/studio/src/api/server.ts` + `packages/studio/src/api/author-sample-writer.ts`
- **行号**：5660–5681（server），138（writer）
- **风险**：`authorId` 未校验即拼接到 `style-library/authors/{authorId}/samples`。
- **修复**：在 server 入口层复用 core 的 ID 规则校验 `authorId`。

### 3. 任意 `x-project-root` 导致路径遍历
- **文件**：`packages/studio/src/api/server.ts`
- **行号**：5689–5850（`/style/authors/:authorId/distillations/*`）
- **风险**：请求头 `x-project-root` 可直接指定任意目录，随后用于 `join(prjRoot, ...)` 读写。
- **修复**：
  ```ts
  function assertProjectRoot(input: string | undefined, serverRoot: string): string {
    const candidate = input ? resolve(input) : resolve(serverRoot);
    const allowed = resolve(serverRoot);
    if (!candidate.startsWith(allowed) && candidate !== allowed) {
      throw new Error("project root out of bounds");
    }
    return candidate;
  }
  ```
