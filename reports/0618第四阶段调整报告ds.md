## 二、关键发现：双实现漂移（🔴 P0 级架构债）

### 2.1 问题本质

Hono 框架使用 **"最后注册胜出"** 的路由匹配策略。当前代码流程：

```
createStudioServer() {
  // 1️⃣ 先注册路由模块（19 个 registerXxxRoutes）
  registerEventsRoutes → GET /api/v1/events  ✅
  registerBooksRoutes  → GET/POST/PUT/DELETE /api/v1/books/*  ✅
  registerSessionsRoutes → GET/POST/PUT/DELETE /api/v1/sessions/*  ← 被覆盖
  ...

  // 2️⃣ 后注册内联路由（server.ts 中 app.get/post/... 调用）
  app.get("/api/v1/sessions", ...)      ← 覆盖了 routes/sessions.ts 的实现
  app.get("/api/v1/sessions/:sessionId", ...) ← 覆盖
  app.post("/api/v1/sessions", ...)     ← 覆盖
  ...
}
```

### 2.2 受影响的重复路由（8 条）

| 路径 | routes/sessions.ts | server.ts 内联 | **实际生效** |
|------|:--:|:--:|:--:|
| `GET /api/v1/interaction/session` | ✅ 注册 | ✅ 注册 | **内联（server.ts）** |
| `GET /api/v1/sessions` | ✅ 注册 | ✅ 注册 | **内联（server.ts）** |
| `GET /api/v1/sessions/:sessionId` | ✅ 注册 | ✅ 注册 | **内联（server.ts）** |
| `POST /api/v1/sessions` | ✅ 注册 | ✅ 注册 | **内联（server.ts）** |
| `PUT /api/v1/sessions/:sessionId` | ✅ 注册 | ✅ 注册 | **内联（server.ts）** |
| `DELETE /api/v1/sessions/:sessionId` | ✅ 注册 | ✅ 注册 | **内联（server.ts）** |
| `POST /api/v1/agent` | ✅ 注册 | ✅ 注册 | **内联（server.ts）** |
| `GET /api/v1/books/:id/analytics` | ✅ 注册 | ✅ 注册 | **内联（server.ts）** |
