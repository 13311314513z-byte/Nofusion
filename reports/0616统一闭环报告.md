## 五、新增发现：P1 级真实缺口（KM/GPT 报告揭示）

### P1-1：Scene Templates 路径错位（GPT 发现，代码确认）

```
API 写入:  story/sources/scene_templates.json  (server.ts 行7074/7095)
Planner 读取: story/state/scene_templates.json   (scene-template-loader.ts 行44)

影响: 用户在前端保存的场景模板永远不会被 Planner 读取。
      前端显示"已保存"，Planner 永远读空数组。
```

### P1-2：Voice Profiles GET/POST 不同步（GPT 发现，代码确认）

```
GET  /voice-profiles   → 读取 story/voice_profiles/index.json
POST /voice-profiles/analyze → 写入 story/voice_profiles/<characterId>.json

影响: POST 分析完成后前端 refetch GET，可能返回空列表。
      Writer 遍历目录读取可用，写作注入不受影响。
```

### P1-3：Plan Alternatives 无用户选择闭环（GPT 发现，代码确认）

```
Core:   planner.ts 已生成 alternatives，runner.ts 已传 generateAlternatives: true
Frontend: write-preview 不返回 alternatives，WriteConfirmPanel 不能选择方案
```

### P2-1：G8 genreTags/contentWarnings 仅前端（三方确认）

如上 §四 G8 证据。

### P2-2：M10 State Changelog 双实现冲突（KM 发现）

```
state-logger.ts     → 写 story/state/state_changelog.jsonl  (本次新增)
state-changelog.ts  → 另一套 changelog 实现 (已有)
两者路径和格式不同，无统一读取端点
```
