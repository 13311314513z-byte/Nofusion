### CLI 测试并发竞态说明

- 默认 `pnpm test`（测试文件并发）在 Windows 下失败，错误为 `Cannot find module packages/cli/dist/index.js`。
- 根因：`publish-package.test.ts` 执行 `npm pack` 触发 `prepack` → `npm run build`，期间会删除并重建 `packages/cli/dist`；其他并发执行的 integration tests 恰好在 `dist` 缺失时访问绝对路径，导致 MODULE_NOT_FOUND。
- 解决方案：顺序执行测试文件后全部通过。

```bash
cd packages/cli
npx vitest run --fileParallelism=false
```

结果：

```text
Test Files  34 passed (34)
     Tests  169 passed (169)
```

> 日志：`packages/cli/0623-cli-sequential-test.log`

```json
{

    "service": "deepseek",
    "services": [
      { "service": "deepseek", "baseUrl": "https://api.deepseek.com", "apiFormat": "chat", "stream": true, "temperature": 0.7 }
    ]
  }
}
```

```bash
inkos book create --title 'Test Book' --genre xuanhuan --platform tomato --target-chapters 5 --chapter-words 1000 --json
```

结果：✅ 成功生成基础设定，Foundation review 81/100 PASSED，返回 `bookId: test-book`。
