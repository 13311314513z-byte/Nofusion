### 4.2 本轮修复

书籍健康度接口最初返回 500：

```text
TypeError: ch.auditIssues is not iterable
```

原因是旧书籍或导入章节可能没有 `auditIssues` 字段，而统计逻辑将其视为必填数组。

修复内容：

1. 将 `auditIssues` 兼容为可选字段。
2. 缺失时按空数组处理。
3. 新增旧章节兼容回归测试。

修复后 `/api/v1/books/test-book-0609/health`、统计和全量 Core 测试均通过。

### 4.3 新建书籍专项

发现并修复一个明确问题：未填写简介时，服务端原本会向 core 传递空字符串 `blurb: ""`，触发最小长度校验并使后台创建立即失败。

修复后：

1. 空简介不再传递 `blurb` 字段。
2. 新增服务端回归测试。
3. HTTP 创建请求不再立即返回简介校验错误。

复核结果：

1. 真实创建任务 `qa-create-234311` 已进入“生成基础设定”阶段。
2. 前 20 秒仍处于 `creating`，但随后成功落盘。
3. 已生成 `book.json`、`story_bible.md`、`style_guide.md`、`outline/story_frame.md`、`outline/volume_map.md` 和多个人物卡。
4. 成功后状态接口返回 404，因为内存中的创建状态已被清理。

判定：空简介缺陷已修复；完整异步创建链路可用。仍建议增加阶段进度、完成态保留时间、总超时和取消入口，否则用户在长任务期间只能看到 `creating`，难以判断是否卡死。

### 6.4 剩余安全风险

`GET /api/v1/services/:service/secret` 当前会把完整 API Key 返回前端。

风险：

1. 浏览器插件、前端日志或 XSS 会扩大 Key 泄露面。
2. 正常页面只需要知道是否已配置，以及编辑时由用户重新输入。

建议改为只返回 `hasApiKey`、掩码尾号或配置状态，禁止向前端返回完整 Key。该项建议按 P1 安全问题处理。
