## 二、核实的高优先级缺陷

### 2.1 `editing || true` 调试残留 🔴

**位置**：`packages/studio/src/pages/ChapterReader.tsx:581`

```typescript
setTimeout(() => { if (editing || true) handleStyleAnalysis(); }, 600);
```

**影响**：`|| true` 使条件永远为真，替换文本后无条件触发风格分析，导致不必要的 LLM 请求和 UI 抖动。

**发现方**：GPT 独有发现，DS/KM 均遗漏。

### 2.2 CI/CD pnpm 版本不一致 🔴

**位置**：`.github/workflows/ci.yml`

```yaml
- uses: pnpm/action-setup@v4
  with:
    version: 9    # ← 硬编码为 9
```

**实际**：`package.json` 声明 `"packageManager": "pnpm@11.5.2"`

**影响**：CI 与本地使用不同 pnpm 版本，lockfile 可能不兼容，构建行为不可预期。

**发现方**：KM 独有发现。

### 2.3 Lint 门禁缺失 🟡

**现状**：
- `pnpm -r lint` 直接报错：无子包定义 `lint` 脚本
- `.eslintrc.json` 为 ESLint v8 legacy 格式
- 无 Prettier 检查脚本

**影响**：代码风格无强制约束，大文件重构风险增加。

### 2.4 Studio API server.ts 单体膨胀 🔴

**规模**：7710 行 / 307KB

**路由数量**：80+ 个 `app.get/post/put/patch/delete` 注册

**风险**：
- 单文件修改 → 全量回归测试
- 路由间共享闭包变量，隐式耦合
- IDE 加载缓慢，代码导航困难
- 新功能添加时无处安放

### 2.5 PipelineRunner 单体膨胀 🔴

**规模**：4100 行 / 165KB

**职责**：建书、续写、审计、修订、导出、导入、状态持久化、通知

**风险**：
- 导入 60+ 依赖
- 各阶段逻辑交织，难以独立测试
- 未来增加新 Agent 管线时无处扩展

### 4.5 不可拆分部分

- **`PipelineRunner` 类本身**：保留为编排器，调用各 stage
- **`runFullPipeline` 方法**：连接各 stage 的主流程

### 6.2 综合裁决

```
概念验证 ──→ 原型 ──→ Alpha ──→ Pre-Beta ──→ Beta ──→ RC ──→ GA
                                              ↑
                                          当前位置
```

**判定：Pre-Beta 末段（v1.5.0）**

1. **功能闭环已达成**。建书→写章→审计→修订→导出全链路可用，46 个 Agent 覆盖完整。
