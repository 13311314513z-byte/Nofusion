[INFERRED] NoFusion 的完美终局是一个**纯编排内核**：PipelineRunner 退化为 ~500 行的薄 facade，所有业务逻辑沉入独立可测试的 `pipeline-*.ts` 模块；Studio 路由层完全模块化，`server.ts` 仅保留工厂函数与中间件注册；测试金字塔以 mock-based 单元测试为主体，真实 LLM E2E 为 opt-in 辅助。

- `style.ts`: ~590 行 → 目标 <600（**已达标** ✅）

### 0.2 约束编译（从语义到语法的硬边界）

#### 🔴 红级约束（违反则程序崩溃或产生严重 Bug）

| 约束 | 语法转化 |
|------|------|
| 禁止 `runner.ts` 与 `pipeline-*.ts` 之间产生循环 import | `import ... from "./runner.js"` 不出现在任何 `pipeline-*.ts` 中 |
| 禁止大段文本替换损坏文件 | `importCanon`/`importChapters` 含 >1000 字符内嵌字符串，不得直接使用 `replace_string_in_file` 整段替换 |
| 禁止类型 re-export 遗漏 `type` 关键字 | `export { TypeA }` → 必须 `export type { TypeA }`（否则 Studio isolatedModules 编译失败） |
| 禁止 pipeline 子模块间产生隐式顺序依赖 | 每个 `pipeline-*.ts` 的函数必须通过显式 deps 接口获取依赖，不得通过模块级闭包共享状态 |

#### 🟡 黄级约束（强烈排斥的反模式）

| 约束 | 语法转化 |
|------|------|
| 禁止测试文件超过 1,500 行 | `wc -l *.test.ts` 硬上限 |
| 禁止已拆模块无独立测试 | 每个 `pipeline-*.ts` 必须有对应的 `pipeline-*.test.ts` |
| 禁止 `any` 类型泛滥 | 全局 `any` 数量从 116 降至 <50 |
| 禁止 server.ts 保留已提取但未删除的死代码 | `grep -r` 逐项验证引用后删除 |
| 禁止路由逻辑跨文件重复 | SSRF helpers / service helpers 必须在 shared 层统一 |

#### 🔵 蓝级约束（完美代码的偏好风格）

| 约束 | 语法转化 |
|------|------|
| 提取方法使用 `.bind(this)` 委托模式 | `private async _foo() { return fooImpl(this.ctx, deps); }` |
| 测试文件使用共享 helper | `import { createRunnerFixture, setupPipelineMocks } from "./pipeline-test-helpers.js"` |
| 新模块 `<200` 行优先 | 超过 200 行的模块应有进一步拆分计划 |

### 0.3 降维打击点

[INFERRED] NoFusion 与普通 LLM 写作工具的核心区别：**不是"能生成小说"，而是"能在工业级约束下可靠地生成"**。普通代码处理 Happy Path（模型正常返回、文件存在、状态一致），NoFusion 的完美终局在以下边界条件下依然优雅降级：

- 模型返回空响应 → 502 `AGENT_EMPTY_RESPONSE` + upstream probe
- 章节状态结算失败 → `state-degraded` 降级保存 + 阻断后续写入
- SQLite 不可用 → Markdown 回退 + 单次警告
- 并发写入 → 资源锁 + reentrancy guard
- 基础设定导入版本冲突 → `expectedRevision` 校验 + 要求重新预览

[COMPUTED] 当前这些降级路径已全部实现。差距在于：部分降级逻辑仍嵌在 `runner.ts` 单体中，未独立测试。

### 2.1 评审建议：当前与完美终局的偏差点

[INFERRED] 三个核心差距：

**差距一：测试先行原则未贯彻**
`pipeline-foundation.ts`(452 行) 被最早拆出，但至今无独立测试。所有 foundation 逻辑仅通过 `pipeline-runner.test.ts` 的 75 个集成测试间接覆盖。这意味着：修改 foundation 逻辑时，必须跑全部 75 个测试（含 LLM mock、文件 I/O、状态同步），反馈循环 ~15 秒，而非独立测试的 ~1 秒。

`pipeline-runner.test.ts`(5,192 行) 和 `server.test.ts`(3,810 行) 是本次推进中几乎未触动的最大技术债。虽然 `pipeline-foundation.test.ts`(6 tests) 已拆出，但主体仍未动。
