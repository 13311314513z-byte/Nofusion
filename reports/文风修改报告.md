## 2. 本次修改详情

### 2.1 `packages/core/src/style-library/aggregate.ts`

**修改内容：** `buildAuthorProfile` 回退路径的 fingerprint 默认值补全新增字段。

**原因：** 当 `profiles.length === 0` 且没有 `existing` 档案时，回退路径生成的 `fingerprint` 只有 9 个旧字段，缺少 `sentenceTypeDistribution` / `paragraphRhythm` / `rhetoricBreakdown` / `dialogueFeatures`。虽然这些字段是可选的（不会导致类型错误），但会导致新建空档案在展示时出现 `undefined`。

**修改前：**
```typescript
fingerprint: {
  dialogueRatio: 0,
  actionDensity: 0,
  // ... 9 个旧字段
  sensoryBreakdown: { visual: 0, auditory: 0, tactile: 0, olfactory: 0, gustatory: 0 },
},
```

**修改后：**
```typescript
fingerprint: {
  dialogueRatio: 0,
  actionDensity: 0,
  // ... 9 个旧字段
  sensoryBreakdown: { visual: 0, auditory: 0, tactile: 0, olfactory: 0, gustatory: 0 },
  sentenceTypeDistribution: { declarative: 0, interrogative: 0, exclamatory: 0, imperative: 0 },
  paragraphRhythm: { shortParaRate: 0, mediumParaRate: 0, longParaRate: 0, lengthHistogram: [] },
  rhetoricBreakdown: { metaphorRate: 0, parallelismRate: 0, personificationRate: 0, hyperboleRate: 0, rhetoricalQuestionRate: 0, repetitionRate: 0 },
  dialogueFeatures: { avgDialogueLength: 0, dialogueFrequency: 0, dialogueTagRatio: {} },
},
```

### 2.2 `packages/core/src/agents/style-fingerprint.ts`

**修改内容：** 移除本地 `StyleFingerprint` 接口定义，改为从 `models/style-profile.js` import 并 re-export。

**原因：** `style-fingerprint.ts` 中定义了一个与 `models/style-profile.ts` 中同名的 `StyleFingerprint` 接口（但不含新字段），而 `core/src/index.ts` 第 303 行从 `style-fingerprint.js` 导出了 `type StyleFingerprint`。这导致了两份定义的存在：
- `models/style-profile.ts` 的 `StyleFingerprint` — 含新字段，是权威定义
- `style-fingerprint.ts` 的 `StyleFingerprint` — 旧定义，仅在模块内部使用

虽然 TypeScript 结构类型系统下两者兼容，但存在维护风险（未来修改 models 时可能忘记同步这里）。

**修改前：**
```typescript
export interface StyleFingerprint {
  readonly dialogueRatio: number;
  // ... 9 个旧字段
}
```

**修改后：**
```typescript
import type { StyleFingerprint } from "../models/style-profile.js";
export type { StyleFingerprint } from "../models/style-profile.js";
```

> **注意：** 初次修改时遗漏了 `export type` 的 re-export，导致 `core/src/index.ts` 的 `export { analyzeStyleFingerprint, type StyleFingerprint } from "./agents/style-fingerprint.js"` 编译失败。已通过在 `style-fingerprint.ts` 中添加 `export type { StyleFingerprint }` 修复。

### 2.3 `packages/studio/src/pages/StyleManager.tsx`

**修改内容：** 在「AI 痕迹检测」Tab 中接入 `RhetoricIssuePanel` 组件，显示修辞重复检测结果。

**原因：** `RhetoricIssuePanel` 组件文件已存在（298 行），但没有任何页面导入和使用。`StyleManager` 的文本分析流程中，当用户上传文件后，会调用 `/style/preprocess/inspect` 获取 `InspectionResult`，其中已包含修辞检测的 `findings`。但此前这些 findings 仅在文本分析 Tab 的底层数据中存在，从未在前端 UI 中展示。

**修改内容：**

1. **新增 import：**
```typescript
import { RhetoricIssuePanel } from "../components/readability/RhetoricIssuePanel.js";
import type { FullStyleDiagnostics, DuplicateRhetoricFinding, RhetoricCategory } from "@actalk/inkos-core";
```

2. **新增类型转换函数：**
```typescript
function convertToRhetoricFindings(findings: ReadonlyArray<InspectionFinding>): ReadonlyArray<DuplicateRhetoricFinding> {
  const RHETORIC_CODE_MAP: Record<string, { category: RhetoricCategory; label: string }> = {
    "duplicate-parallelism": { category: "parallelism", label: "排比句式" },
    "duplicate-metaphor": { category: "metaphor", label: "比喻手法" },
    // ... 其余 8 类
  };
  return findings
    .filter((f) => f.code.startsWith("duplicate-"))
    .map((f) => ({
      id: f.findingId ?? `...`,
      category: mapped?.category ?? "repetition",
      label: mapped?.label ?? f.messageKey,
      count: f.count,
      perThousandChars: f.perThousandChars ?? 0,
      severity: f.rhetoricSeverity ?? "low",
      confidence: f.confidence ?? 0.5,
      examples: f.samples.map((text, i) => ({ text, lineNumber: f.lineNumbers?.[i] ?? 0 })),
      ranges: f.ranges ?? [],
    }));
}
```

3. **修改 detection tab 渲染：**
```tsx
{activeTab === "detection" && (
  <div className="max-w-4xl mx-auto py-4 space-y-6">
    <AITellsPanel t={t as unknown as (key: string) => string} />
    {inspectionResult && inspectionResult.findings.filter((f) => f.code.startsWith("duplicate-")).length > 0 && (
      <div className="border border-border rounded-lg p-4 bg-card">
        <h3 className="font-semibold text-sm mb-3">
          {t("style.rhetoricDetection") ?? "修辞重复检测"}
        </h3>
        <RhetoricIssuePanel
          findings={convertToRhetoricFindings(inspectionResult.findings)}
          mode="full"
          actions={["highlight", "ignore"]}
          storageKey="style-rhetoric"
        />
      </div>
    )}
  </div>
)}
```

**设计决策：**
- 不新增独立 Tab，而是在现有「AI 痕迹检测」Tab 下方堆叠显示，减少 UI 改动
- 只有当 `inspectionResult` 存在且包含修辞类 finding 时才渲染，避免空状态
- 将容器宽度从 `max-w-2xl` 扩展到 `max-w-4xl`，为问题面板提供更充裕的展示空间
