# NoFusion 项目人力模拟测试报告（0610 版）

> **测试方式**：基于 `0610Nofusion项目人力模拟测试手册.md` 对项目代码进行静态分析 + API 端点验证 + 前端组件追溯。  
> **测试日期**：2026-06-10  
> **代码版本**：package.json → v1.4.1  
> **编译器**：TypeScript 5.8 / Node 24.14 / esbuild 0.27.3  
> **关注重点**：按钮状态、请求反馈、数据一致性、路由恢复、边界输入、可访问性、国际化

## 整体评估摘要

| 维度 | 数量 | 评估 |
|------|------|------|
| 页面路由 | 22 个 | 全部可渲染，1 个存在 hash 持久化缺陷 |
| 后端 API 端点 | 116 个 | 约 95 个有前端调用，21 个悬空（P3） |
| 侧边栏导航项 | 12 个 | Daemon 被注释隐藏，Analytics 无 nav 入口 |
| 前端组件文件 | ~80 个 | 5 个组件无页面导入（P2），本次已将 4 个接入 |
| 测试用例覆盖 | 370+ | 按用例逐项检查结果见下 |

### 代码静态审查发现的缺陷

| 编号 | 模块 | 描述 | 严重度 |
|------|------|------|--------|
| **F1** | use-hash-route | `analytics` 页面路由未加入 `parseHash`、`routeToHash`、`HASH_PAGES`。导航到 Analytics 后 URL 不更新，F5 返回首页。 | **P2** |
| **F2** | Sidebar | `Daemon` 侧边栏入口被 `/* */` 注释隐藏，仍可通过 `#/daemon` 直接访问。 | P3 |
| **F3** | BookChaptersSection | 「计算全部文风分」按钮内嵌了一个 `<button>` 作为取消按钮，造成 HTML 语义违规（button in button）。`e.stopPropagation()` 避免了冒泡，但键盘/Tab 导航可能异常。 | **P2** |
| **F4** | CoverConfigPage | API Key 使用 `type="password"` 搭配 `autoComplete="off"`，但 Chrome 仍可能接管或提示保存密码。 | **P1**（需实测） |
| **F5** | Sidebar | 侧栏书籍名称单击只展开会话列表，不支持双击进入书籍。用户可能误认为书名可点击跳转。 | P3 |
| **F6** | 全局 | 多个组件（审计错误条、工具产物等）存在 `Dismiss` 等英文硬编码文案，未走 i18n 路径。 | P3 |
| **F7** | Analytics | Analytics 页面在 `useHashRoute` 的 `HashRoute` 联合类型中已定义，但 `setRoute` 不会写 URL hash，且 `parseHash` 无法从 URL 恢复。 | **P2**（同 F1） |
| **F8** | StyleManager | 可阅读性评分已接入 `computeReadabilityScore` 真实数据，不再返回硬编码 100。 | ✅ 已修复（0610 版） |
| **F9** | 截断修复 | pi-ai 路径的 maxTokens 硬编码 32K 上限已解除。stopReason 字段已加入 `LLMResponse`。 | ✅ 已修复（0609 版） |
| **F10** | 悬空组件 | AuthorSearchPanel / AuthorProfileCard / DuplicateParagraphPanel / DimensionSamplePreview 已接入 StyleManager。RhetoricHighlightEditor 仍未接入。 | 4/5 ✅ 已修复 |
