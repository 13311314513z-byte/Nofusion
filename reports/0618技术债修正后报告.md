## 五、已修正项目技术细节

### 5.1 写作安全（P0-1）

```
write-next/draft/rewrite 端点:
  ├── acquireWriteJob() → 幂等键防重复 → 409 WRITE_IN_PROGRESS
  ├── withPipeline()    → 10min 超时 → timed_out 状态
  └── /write-status     → SSE 断连后查询接口
```

### 5.2 前端请求分级超时（P0-3）

| 操作类型 | 超时 | 应用端点 |
|------|:--:|------|
| CRUD | 30s（默认） | books GET/PUT/DELETE 等 |
| 写作 | 300s | revise/resync/audit |
| 导入 | 600s | radar/scan, import/foundation |

### 5.3 性能优化（P1）

| 优化 | 效果 |
|------|------|
| config mtime 缓存 ×2 | 消除每次请求的磁盘 I/O + JSON.parse |
| agentClients TTL | 过期连接自动回收，防止泄漏 |
| detect-all 分批 | 100章 → 10章/批，防止 OOM |
| scanRoles 并行 | 9目录串行 → 并行，I/O 时间降为 1/9 |
| computeDiff Set | O(n²) → O(n)，大量行时显著加速 |
| regex 缓存 | 避免每次调用编译正则 |

### 5.4 防御性措施（P2）

| 措施 | 说明 |
|------|------|
| Body limit 10MB | 防止恶意 JSON OOM，413 响应 |
| sessions 分页 | 默认 50 条/页，上限 200，含 total |
| perf 监控 | 2s 阈值 warn，便于发现性能退化 |
| ETag | Dashboard 支持 304，减少重复序列化 |
| 结构化日志 | `INKOS_STRUCTURED_LOGS=1` 切换 NDJSON |
