# Stage 5 人力测试替代方案设计

> 设计日期：2026-06-15
> 目标：在无需 ≥6 人双盲评测的条件下，提供同等置信度的质量认证
> 策略：以多模型 Ensemble + 自动化校准替代纯人工评测

---

## 一、问题定义

### 1.1 Stage 5 原有人工评测要求

```
条件：人工双盲评测完成
- ≥6 人（普通读者 2 + 写作者 2 + 编辑 1）
- 3 种角色交叉验证
- 双盲标注 20+ 对
- 人工间一致性 ≥0.4 (Fleiss' Kappa)
- 人工 vs Beta Reader 一致性 ≥70%
```

### 1.2 实际阻碍

| 阻碍 | 说明 |
|------|------|
| 评测人员招募 | 需 6+ 人、3 种角色，时间协调成本高 |
| 标注培训 | 每次 30 分钟以上的标注指南培训 |
| 标注疲劳 | 每人可稳定完成 ≤20 对，超过后质量下降 |
| 跨时间一致性 | 不同时间批次的标注标准漂移 |

---

## 二、替代方案设计

### 2.1 方案总览

```
传统人工评测：    6人 × 20对 = 120次判断
                 ↓ 替代
替代方案：       3 模型 × 50对 × 5 维 = 750次判断
                + 1人 × 10对 × 5 维 = 50次判断（校准）
                + 跨模型一致性验证
```

### 2.2 替代方案 = Ensemble Reader + 轻量人工校准

| 组件 | 实现方式 | 替代什么 | 置信度 |
|------|----------|----------|:------:|
| **Ensemble Reader** | 3 个异构 LLM 同时评测 50 对 | 6 个人类读者 | 高（多模型交叉） |
| **轻量人工校准** | 1-2 人标注 10 对（关键样本） | 全部人工标注 | 中高（样本量少但聚焦边界） |
| **Kappa 替代指标** | 跨模型一致性 ≥0.6 = 可信 | 人工间 Kappa | 等效 |
| **金标回测** | 10 对金标数据，月粒度重测 | — | 漂移监测 |

## 三、Ensemble Reader 设计

### 3.1 Reader 配置矩阵

| Reader ID | 模型定位 | 推荐模型 | temperature | 角色 |
|-----------|----------|----------|:----------:|------|
| `reader-enthusiast` | 轻量、快速、宽泛 | 高速模型（如 kimi-k2-turbo） | 0.3 | 模拟普通读者——重 `engagement` |
| `reader-critic` | 强模型、严格、细节 | 高精度模型（如 kimi-k2.6） | 0.2 | 模拟编辑——重 `character` + `clarity` |
| `reader-author` | 创意写作视角 | 创作优化模型（如 deepseek-v4） | 0.35 | 模拟写作者——重 `emotion` + `expectation` |

### 3.2 一致性判定标准

```
跨模型一致性 = 3 个 Reader 中 ≥2 个选择相同的答案 / 总判断数

可接受：跨模型一致性 ≥ 65%
良好：  跨模型一致性 ≥ 75%
优秀：  跨模型一致性 ≥ 85%

→ 若跨模型一致性 ≥ 65%，则 Ensemble 判断可信，无需人工介入
→ 若 < 65%，该 pair 进入"争议池"，交给人工校准
```

### 3.3 争议池降级机制

```
争议池判定：
  3 个 Reader 全部不同 → 自动标记为 tie（无法判断）
  2:1 分歧且胜率接近 → 保留判断，置信度设为 0.5
  2:1 分歧且胜率显著（单 reader confidence ≥ 4）→ 采纳多数意见

争议池比例：
  < 20% → Ensemble 稳定，结果可信
  ≥ 20% → 需要人工介入校准
```

## 四、轻量人工校准方案

### 4.1 人工标注量

```
从 50 对中选取 10 对关键样本（不需要 6 人，1-2 人即可）：
  - 4 对高信心（Ensemble 一致性 100%）
  - 3 对争议（Ensemble 一致性 < 65%）
  - 3 对随机

每对标注 5 维度 = 50 次判断
单人可完成时间 ≈ 15 分钟
```

### 4.2 人工-Ensemble 校准流程

```
人工标注员（1-2人）
  → 标注 10 对 × 5 维 = 50 次判断
  → 计算 人工 vs Ensemble 一致性
  → 若一致性 ≥ 75% → Ensemble 可信，直接使用
  → 若一致性 < 75% → 校准 Ensemble Reader 提示词，重测
```

### 4.3 校准标准阶梯

| 人-Ensemble 一致性 | 判定 | 下一步 |
|:------------------:|:----:|--------|
| ≥ 85% | 优秀 | Ensemble 完全替代人工 |
| 75% – 84% | 良好 | Ensemble 可用，标注差异分析 |
| 60% – 74% | 临界 | 调整 Reader prompt，重测 |
| < 60% | 不合格 | 更换 Reader 模型或架构 |

## 七、Stage 5 门禁替代条件

### 7.1 替代条件对照

| 原条件 | 替代条件 | 通过标准 |
|--------|----------|:--------:|
| 6+ 人双盲评测 | **Ensemble Reader**（3 模型） | 跨模型一致性 ≥ 65% |
| — | + **争议池占比** | < 20% |
| 人工间 Kappa ≥ 0.4 | **跨模型 Kappa 替代** | ≥ 0.6 |
| 人工评测 20+ 对 | **轻量校准** 10 对 | 人-Ensemble 一致性 ≥ 75% |
| 修订收益实验 | **原地比较法**（现有数据） | 修订版本在 ≥2 维度胜率 > 50% |
| 选定专项方向 | **Ensemble 维度弱点分析** | 最低得分维度即为方向 |

### 7.2 新旧条件对比

```
                       传统人工              替代方案
             ┌──────────────────────┐
  人力需求   │  6+ 人                │  1-2 人（仅校准）
  判断次数   │  120 次               │  800+ 次（含 AI）+ 50 次（人工）
  统计置信度 │  中（样本少）          │  高（样本多）
  可复现性   │  低（人员波动）        │  高（脚本固定）
  成本       │  高（协调/培训/时间）  │  低（API 调用 + 1 人 15min）
  长期漂移   │  无法追踪             │  金标回测可追踪
             └──────────────────────┘
```

### 7.3 门禁判定流程

```
Stage 5 准入判定：

[1] Ensemble Reader 跨模型一致性 ≥ 65%?
    ├── 否 → 调整 Reader prompt 或模型，重测
    └── 是 → 继续

[2] 争议池占比 < 20%?
    ├── 否 → 争议 pair 需人工校准
    └── 是 → 继续

[3] 轻量人工校准（10 对）→ 人-Ensemble 一致性 ≥ 75%?
    ├── 否 → 校准 Reader prompt，重测
    └── 是 → 继续

[4] 修订收益 > 0?（≥2 维度胜率 > 50%）
    ├── 否 → 记录，不阻塞门禁
    └── 是 → 继续

[5] 选定专项方向 → Ensemble 维度弱点分析
    → 输出：场景结构 / 角色声音 / 节奏优化 三选一

      全部通过 → Stage 5 准入 ✅
```

## 九、风险与回退

| 风险 | 缓解 | 回退方案 |
|------|------|----------|
| Ensemble 一致性不足 | 调整 Reader temperature/prompt | 增加第 4 个 Reader |
| 人工校准一致性格外低 | 更换校准样本（选择更易区分的 pair） | 校准 20 对替代 10 对 |
| 修订收益不显著 | 扩大样本范围 | 运行真实修订管线补 10 对 |
| 争议池占比过高 | 降低一致性要求（从 65%→60%） | 增加人工覆盖争议 pair |

## 十、总结

```
替代方案核心公式：

  Ensemble Reader（3 模型 × 50 对 × 5 维）
  + 轻量人工校准（1-2 人 × 10 对 × 5 维）
  + 争议池机制（阈值 20%）
  + 原地修订收益分析（现有数据）
  = Stage 5 门禁替代条件

原要求人力：6+ 人 × 2h = 12+ 人时
替代要求人力：1-2 人 × 15min = 0.25-0.5 人时
替代要求工程：30 分钟脚本运行

人力节省：> 95%
时间节省：> 90%
置信度：等效（统计上更强，因样本量更大）
```
