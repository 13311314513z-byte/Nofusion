## 二、模块逐项测试与扩展建议

### 模块 1：项目与配置管理

#### 当前功能
| 功能 | 状态 | 说明 |
|------|------|------|
| 项目初始化 (`inkos init`) | ✅ | CLI 与 Studio 均支持 |
| 项目配置读取/更新 | ✅ | `GET/PUT /project` |
| LLM 服务商配置 | ✅ | 8 种 provider + custom |
| 模型覆盖配置 | ✅ | `GET/PUT /project/model-overrides` |
| 通知配置 | ✅ | webhook + 安全校验 |
| 审计配置 | ✅ | provider/模型配置 |
| 语言切换 | ⚠️ | 功能存在，但部分 UI 文案未走 i18n |

#### 测试结果
- Core：`init` 相关逻辑覆盖充分
- CLI：`inkos init`、`config set/show` 测试通过
- Studio：配置面板测试通过

#### 存在问题
1. **CLI `doctor` 超时测试失败**：localhost OpenAI-compatible 端点探测逻辑在严格预算下仍超时
2. **配置分散**：`targetChapters`/`chapterWordCount` 在 `PUT /books/:id`，其他配置在 `PATCH /books/:id/config`
3. **无配置模板**：新建项目无题材/体裁模板

#### 扩展建议
| 优先级 | 建议 | 预期价值 |
|--------|------|----------|
| P1 | 修复 `doctor` 超时测试，或增加 `--skip-connectivity` 默认开启 | 消除唯一测试失败 |
| P2 | 统一书籍配置入口为 `BookSettingsSection` | 降低用户认知负担 |
| P2 | 增加项目级创作模板（短篇/中篇/长篇/剧本） | 提升新建效率 |
| P3 | 配置变更历史与回滚 | 增强安全感 |
