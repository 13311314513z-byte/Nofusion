## 六、当前最高风险节点

### 1. Studio server 编译失败

这是当前第一优先级。只要 `api-utils.ts` 还被 server 编译链路引用并包含 `window`，Studio build 就不能通过。

建议先拆文件边界，不要扩大重构：

- `shared/api-utils.ts` 只保留 server/client 均可用的纯函数和类型；
- 浏览器缓存失效、`window.location`、错误复制等逻辑放回 `hooks/` 或 `client/` 专用模块；
- 必要时把 `error-copy.ts` 纳入 client tsconfig，而不是让 server tsconfig 被迫包含 UI 辅助。

### 2. PipelinePool 初始化破坏测试基座

当前 `createStudioServer` 创建时立即初始化 pool，导致大量既有测试在 mock 层失败。

建议：

- 先让测试 mock 补齐 `PipelinePool`，或改成 partial mock；
- 更稳妥的是把 pool 延迟到 `withPipeline` 首次使用时初始化，减少 server 创建副作用；
- 为 `PipelinePool` 加一组独立单测，再恢复 Studio server 测试。

### 3. Agent 外部编辑功能被 stub

`routes/agent.ts` 当前 shadow 了导入的 `tryHandleExternalChatEdit`，这会让“外部聊天编辑快捷路径”失效。

建议立即移除本地 stub，恢复调用 `shared/agent-helpers.ts` 的真实实现，并补一个 `/api/v1/agent` 外部章节编辑回归测试。

### 4. Foundation/legacy 行为回归

Core 当前两个失败测试都指向基础设定文件写入和 revise 备份行为：

- 空 `storyFrame` 时 legacy fallback 不应生成 `outline/story_frame.md`；
- legacy revise 时应创建 `.backup-*` 并保留旧文件备份。

这类回归会影响老项目兼容和 Phase 5 迁移安全，不能放到后面处理。

### 5. C3 lint 硬失败

`useStyleManagerState.ts` 两条 `react-hooks/exhaustive-deps` disable 注释在当前 eslint 配置下是错误。这里不需要大改，先删除无效注释或引入并配置 `eslint-plugin-react-hooks`，再处理依赖数组本身。
