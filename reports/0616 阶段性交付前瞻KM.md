### 2.1 CLI 集成测试失败根因

`cli-integration.test.ts` 通过 `pnpm --dir <tempProject> exec node <cli-dist>` 启动子进程，子进程在解析 `@actalk/inkos-core` 时无法通过 workspace symlink 定位到 `packages/core/dist/index.js`，报错：

```
Error [ERR_MODULE_NOT_FOUND]: Cannot find module
'C:\Users\white\Downloads\Nofusion-main\packages\cli\node_modules\@actalk\inkos-core\dist\index.js'
imported from C:\Users\white\Downloads\Nofusion-main\packages\cli\dist\utils.js
```

**影响**：
- 根级 `pnpm test` 无法通过，直接阻断 CI 绿通。
- 影响 CLI 作为独立 npm 包发布后的可安装性验证。
- 该问题在 Linux/macOS 下可能不重现，导致跨平台回归遗漏。

## 四、关键风险点

### 🔴 P0 风险：CLI 集成测试在 Windows workspace 下失败

详见 2.1。这是当前唯一导致根级测试失败的阻塞项。

### 🟡 P1 风险：单体文件持续膨胀

- `studio/api/server.ts` 7709 行：所有路由、业务逻辑、LLM 调用、文件操作集中在一处。
- `core/pipeline/runner.ts` 4112 行：建书/续写/审计/修订/导出/导入全塞在一个文件。

### 🟡 P1 风险：Lint 与代码风格门禁缺失

- `pnpm -r lint` 直接报错：`None of the selected packages has a "lint" script`。
- `.eslintrc.json` 是 ESLint v8 格式，项目已升级到 ESLint v9。
- 无 Prettier 检查脚本。

### 🟡 P1 风险：CI 与本地工具链不一致

- `packageManager` 声明 `pnpm@11.5.2`，CI 中硬编码 `version: 9`。
- CI 无 `lint` job，无构建产物缓存，5 个 job 各自重复 `pnpm install`。

### 🟢 已修复（相对于 0616 早期报告）

- `cleanStringArray` 空数组语义已修正，可正常清空字段。
- `SceneTemplateRecord` 已与 `SceneTemplateSchema` 字段对齐，Planner 可正常消费。
- Studio `StyleImportTab` 等 typecheck 错误已修复。

## 六、发展建议

### 6.1 近期（1–2 周）：先打通交付门禁

| 优先级 | 任务 | 验收标准 |

### 6.2 中期（2–4 周）：偿还架构债务

| 优先级 | 任务 | 验收标准 |
