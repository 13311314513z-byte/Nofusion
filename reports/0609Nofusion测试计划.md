# 0609 NoFusion 人力测试计划

> **目标**：从"新建书籍"开始，逐模块验证 InkOS Studio 全部核心功能的完整性与正确性。
> **测试方式**：纯人工操作，覆盖正向流程 + 边界情况。
> **预计耗时**：约 2.5–3.5 小时（完整遍历）；核心路径约 1 小时（批次 A）
> **版本基线**：@actalk/inkos v1.4.1 + 0608 全部 Batch A/B/C 改动

---

## 测试环境准备

### 前置条件

- [ ] Node.js ≥ 20、pnpm 已安装
- [ ] 项目依赖已安装（`pnpm install`）
- [ ] Core 包已构建（`pnpm --filter @actalk/inkos-core exec tsc`）
- [ ] API Key 已配置（`~/.inkos/.env` 或项目 `.env`，含 `INKOS_LLM_API_KEY`）
- [ ] Studio 开发服务器可启动（`pnpm --filter @actalk/inkos-studio dev`）

### 启动命令

```bash
# 终端 1：API 服务
cd packages/studio && pnpm run dev:server

# 终端 2：前端（Vite）
cd packages/studio && pnpm run dev:client
```

### 测试账号/数据

- 准备一段 500–2000 字的中文叙事文本（测试用）
- 准备一段 500–2000 字的英文文本（测试多语言）
- 准备一个 `.txt` 文件 + 一个 `.md` 文件（测试文件导入）

### 测试记录规范

- **每个 P0/P1 缺陷**：必须附截图 + 浏览器 Console 报错信息
- **文件命名**：`缺陷_<模块>_<编号>_<简述>.png`
- **日志收集**：`packages/studio/logs/` 下最近 3 个日志文件
- **数据验证**：涉及配置/保存的操作，检查 `inkos.json` 结构变化
