## 二、Phase 0：安全与基础设施（1 天）

### 2.1 API Key 安全加固（P1-5）

| 属性 | 内容 |
|------|------|
| **问题** | `GET /api/v1/services/:service/secret` 返回完整 API Key |
| **影响** | 浏览器插件、XSS、前端日志均可泄露 Key |
| **方案** | 后端仅返回 `{ hasApiKey: boolean, keyPreview: string }`（掩码末4位） |
| **涉及文件** | `server.ts` line 2802（GET endpoint） |
| **前端适配** | `ApiKeyInput.tsx` 已支持编辑时用户重新输入，只需确保「已配置」状态下不通过 GET 回显完整 Key |
| **测试** | 后端端到端测试 + 前端掩码渲染验证 |
| **工作量** | 0.3 天 |

### 2.2 会话端点 404 修复（P1-11）

| 属性 | 内容 |
|------|------|
| **问题** | `GET /api/v1/books/{id}/sessions` 返回 404 |
| **影响** | 侧栏无法加载书籍会话列表 |
| **排查方向** | 1. 确认路由注册路径（`server.ts` 中是否注册此端点）<br>2. 确认前端调用路径（`book-session-store.ts` 中 `listBookSessions` 的 API 路径）<br>3. 确认 Hono 路由匹配规则（参数中的 `/` 是否需要 `{.+}` 后缀） |
| **工作量** | 0.2 天 |

### 2.3 文件锁 TOCTOU 残余窗口（P1-1）

| 属性 | 内容 |
|------|------|
| **现状** | 已增加重读校验缩小窗口，但 `unlink` 前仍有竞争 |
| **方案** | 使用 `rename` 原子操作替代「读-删-重建」模式：以随机后缀创建临时锁文件 → `rename` 到目标路径 |
| **涉及文件** | `state/manager.ts` |
| **测试** | 新增并发锁竞争测试 |
| **工作量** | 0.5 天 |

## 十一、否决与暂缓建议

以下功能需求经评估后建议暂缓或否决：

| 需求 | 来源 | 否决理由 |
|------|------|----------|
| 全量 API Key 加密存储 | 安全建议 | 当前 OS 文件权限 + 项目隔离已足够；全量加密需引入密钥管理，复杂度高 |
| 移动端适配 | 功能建议 | 当前 UI 为桌面端设计，移动端适配需重新布局，工作量 > 5 天且非核心场景 |
| 多语言翻译（非中/英） | 国际化 | 当前仅中/英双语，增加第三方语言维护成本高，且无明确用户需求 |
| 第三方登录/OAuth | 安全建议 | 当前为本地单用户应用，OAuth 需后端改造，复杂度远高于收益 |
| D3.js 图表库引入 | 可视化 | 当前纯 CSS/SVG 图表已满足需求；引入 D3 增加 100KB+ bundle 体积 |
