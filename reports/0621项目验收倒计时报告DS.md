### 0.1 第一性原理（终局方向）

[INFERRED] NoFusion 的完美终局是**纯编排内核 + 全模块化路由 + 完备测试金字塔**：

- `runner.ts` 退化为 ~500 行 facade，所有业务逻辑在独立 `pipeline-*.ts` 模块中，每个模块有独立单元测试

### 0.3 降维打击点

[INFERRED] NoFusion 与普通 LLM 写作工具的核心区别：**不是"能生成小说"，而是"能在工业级约束下可靠地生成"**。

[COMPUTED] 当前已实现的降级路径（全部通过验证）：
- 模型返回空响应 → 502 `AGENT_EMPTY_RESPONSE` + upstream probe ✅
- 章节状态结算失败 → `state-degraded` 降级保存 + 阻断后续写入 ✅
- SQLite 不可用 → Markdown 回退 + 单次警告 ✅
- 并发写入 → 资源锁 + reentrancy guard ✅
- 基础设定导入版本冲突 → `expectedRevision` 校验 + 要求重新预览 ✅

[COMPUTED] 差距：降级逻辑仍嵌在 `runner.ts` 单体中，未独立测试，未形成可观测的降级事件流。
