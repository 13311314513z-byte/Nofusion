### P0-3 CLI 集成测试 pnpm workspace 不兼容

**状态：✅ 已修复，169 测试全绿**

合并报告判定 CLI 集成测试子进程模块解析与 pnpm workspace 不兼容，为三方共识 P0。

本轮核验发现实际问题叠加三层：
1. `execFileSync("pnpm.cmd", ...)` 在 Node.js ≥18.20 的 Windows 环境下抛 `spawnSync pnpm.cmd EINVAL`；
2. 测试使用临时目录作为 cwd，pnpm exec 要求该目录存在 `package.json`；
3. 即便使用 `shell: true`，cmd.exe 的参数拼接会导致含空格的值（如 `--title "stale book"）被解析为位置参数，触发 `too many arguments for 'create'`。

**修复方案：**
- 通过 `node <pnpm-global>/node_modules/pnpm/bin/pnpm.mjs` 直接调用 pnpm，绕过 `.cmd` 的 EINVAL 限制；
- 使用 `--dir <projectDir>` 指定工作目录，避免 `shell` 参数；
- 在 `beforeAll` 中为临时项目目录写入 `package.json`，满足 pnpm exec 的前置条件。

实测结果：

```bash
pnpm --filter @actalk/inkos test
# Test Files  34 passed (34)
# Tests       169 passed (169)
```

**结论：** CLI 集成测试已稳定通过，P0-3 闭合。

## 四、第二阶段新发现的阻断回归与修复

在核验过程中，合并报告之后的新 commit 引入了以下回归，均已修复：

### 4.1 Studio 生产构建失败：`recharts` 未安装却被声明为 manual chunk

**严重度：P0（阻塞 Studio 构建与 CLI publish-package 测试）**

`packages/studio/vite.config.ts` 中声明了：

```ts
manualChunks: {
  "vendor-react": ["react", "react-dom"],
  "vendor-lucide": ["lucide-react"],
  "vendor-charts": ["recharts"],   // ← 未安装依赖
}
```

但 `recharts` 既未在 `packages/studio/package.json` 中出现，也无任何源码引用。`vite build` 因此抛出：

```
Could not resolve entry module "recharts".
```

**修复：** 移除未使用的 `vendor-charts` manual chunk。

```bash
pnpm --filter @actalk/inkos-studio build
# Build artifacts verified
```

### 4.2 Core 新增测试数据与 schema 不匹配

`packages/core/src/__tests__/p2-agent-smoke.test.ts` 中 `RelationshipDeltaSchema` 测试数据使用字段 `cause` 及不存在字段 `intensityChange`/`mutual`，与 `event-chain.ts` 中 `trigger` 必填字段冲突，导致 Core 全量测试失败。

**修复：** 调整测试数据，增加 `{ timeout: 30000 }` 避免 forks 池下动态导入 `@actalk/inkos-core` 超时。

### 4.3 `state-changelog-evidence.test.ts` 全量运行时超时

该测试动态导入整个 `@actalk/inkos-core` 包。在 `pool: "forks"` 全量运行时，默认 5s 超时不足。

**修复：** 为关键测试用例设置 `timeout: 30000`。
