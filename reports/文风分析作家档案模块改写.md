### 1.3 代码审查发现的关键问题

在现有代码基础上深入审查后，发现以下**必须先于新功能开发解决的基础问题**：

#### 1.3.1 `analyzeStyle` 与 `detectDuplicateRhetoric` 存在重复检测

`style-analyzer.ts` 中定义了独立的 `RHETORICAL_PATTERNS`：

```typescript
// 实际代码 (style-analyzer.ts:10-16)
const RHETORICAL_PATTERNS = [
  { name: "比喻(像/如/仿佛)", regex: /[像如仿佛似](?:是|同|一般|一样)/g },
  { name: "排比", regex: /[，。；]([^，。；]{2,6})[，。；]\1/g },
  { name: "反问", regex: /难道|怎么可能|岂不是|何尝不/g },
  { name: "夸张", regex: /天崩地裂|惊天动地|翻天覆地|震耳欲聋/g },
  { name: "拟人", regex: /[风雨雪月花树草石](?:在|像|仿佛).*?(?:笑|哭|叹|呻|吟|怒|舞)/g },
  { name: "短句节奏", regex: /[。！？][^。！？]{1,8}[。！？]/g },  // 非修辞项，用于标记节奏
];
```

而 `semantic-duplication.ts` 中已有更完整、更精确的 10 类修辞检测（含聚类分析、置信度计算、位置范围等）。两套检测对同一文本可能给出**不一致的结果**，导致：
- `StyleProfile.rhetoricalFeatures` 和 `DuplicateRhetoricResult` 数据冲突
- 用户困惑：文风分析显示"比喻 3 处"，语义查重显示"比喻 6 处"

**修复建议（P0）：** 在 `analyzeStyle` 中移除独立的 `RHETORICAL_PATTERNS`，改为调用 `detectDuplicateRhetoric()` 获取修辞数据，然后转换为 `rhetoricalFeatures` 格式：

```typescript
import { detectDuplicateRhetoric } from "../utils/semantic-duplication.js";

// 替换原有的 RHETORICAL_PATTERNS 循环
const rhetoricResult = detectDuplicateRhetoric(text, "zh");
const rhetoricalFeatures = rhetoricResult.findings
  .filter((f) => f.count >= 2)
  .map((f) => `${f.label}(${f.count}处)`);
```

#### 1.3.2 `fetchUrl` 的正则去标签过于简单

当前 `fetchUrl()`（`web-search.ts:73-78`）使用简单的正则去除 HTML 标签：

```typescript
return text
  .replace(/<script[\s\S]*?<\/script>/gi, "")
  .replace(/<style[\s\S]*?<\/style>/gi, "")
  .replace(/<[^>]*>/g, " ")
  .replace(/\s+/g, " ")
  .trim()
  .slice(0, maxChars);
```

**问题：**
- 对现代 SPA（React/Vue 渲染的内容）无效，抓取到的可能是 JS 代码而非正文
- 没有优先提取 `<main>` / `<article>` 等语义标签的内容
- 导航栏、页脚、广告等噪声内容没有被过滤
- 中文网页常见的 `<div class="content">` 等结构无法识别

**修复建议（P1）：** 引入轻量 HTML 解析器提取正文：

```typescript
// 方案 A：使用 cheerio（推荐，已广泛验证）
import * as cheerio from "cheerio";

function extractArticleText(html: string): string {
  const $ = cheerio.load(html);
  // 优先选择语义标签
  const selectors = ["article", "main", "[role='main']", ".content", "#content", ".post-content", ".entry-content"];
  for (const sel of selectors) {
    const el = $(sel).first();
    if (el.length && el.text().length > 500) {
      return el.text().replace(/\s+/g, " ").trim();
    }
  }
  // 降级：取 body 中段落最多的区域
  return $("body").text().replace(/\s+/g, " ").trim();
}
```

> 若不想引入新依赖，可使用方案 B：基于段落密度的启发式算法（统计 `<p>` 标签密度最高的 `<div>` 区域）。

#### 1.3.3 `mergeStyleProfiles` 聚合逻辑需同步扩展

`aggregate.ts` 的 `mergeStyleProfiles()` 对 `StyleFingerprint` 的每个字段逐个做加权平均。如果新增 `ExpandedFingerprint` 字段（如 `sentenceTypeDistribution`、`paragraphRhythm`、`rhetoricBreakdown` 等），必须在 `mergeStyleProfiles` 中同步添加聚合逻辑，否则新字段在多来源聚合时会丢失。

**影响评估：** 这是结构性问题。若只扩展 `StyleFingerprint` 接口而不更新 `mergeStyleProfiles`，则：
- 单来源档案正常（直接复制）
- 多来源档案的新字段值为 `undefined` 或默认值

**修复建议：** 将 `mergeStyleProfiles` 改为基于字段遍历的通用聚合，减少未来扩展时的遗漏：

```typescript
function mergeNumericField(
  profiles: ReadonlyArray<StyleProfile>,
  weights: ReadonlyArray<number>,
  extractor: (p: StyleProfile) => number,
): number {
  const totalWeight = weights.reduce((a, b) => a + b, 0);
  if (totalWeight === 0) return 0;
  let sum = 0;
  for (let i = 0; i < profiles.length; i++) {
    sum += extractor(profiles[i]) * (weights[i] / totalWeight);
  }
  return Math.round(sum * 100) / 100;
}

// 聚合 fingerprint 时统一处理
const mergedFingerprint: StyleFingerprint = {
  dialogueRatio: mergeNumericField(profiles, weights, p => p.fingerprint.dialogueRatio),
  actionDensity: mergeNumericField(profiles, weights, p => p.fingerprint.actionDensity),
  // ... 所有现有字段
  // 新增字段自动参与聚合（若 extractor 返回 undefined 则默认为 0）
  sentenceTypeDistribution: {
    declarative: mergeNumericField(profiles, weights, p => p.fingerprint.sentenceTypeDistribution?.declarative ?? 0),
    // ...
  },
};
```

#### 3.1.2 搜索策略

```typescript
// packages/studio/src/api/author-search.ts

interface AuthorSearchConfig {
  readonly authorName: string;
  /** 搜索语言 */
  readonly language: "zh" | "en";
  /** 每个来源最大结果数 */
  readonly maxResultsPerSource?: number;
}

interface SearchSourceResult {
  readonly source: "tavily" | "bing" | "google";
  readonly title: string;
  readonly url: string;
  readonly snippet: string;
  readonly relevance: number; // 0-1
}

/**
 * 多来源搜索作家原文。
 * 优先使用 Tavily（已集成），用户可配置额外来源。
 */
export async function searchAuthorWorks(
  config: AuthorSearchConfig,
): Promise<ReadonlyArray<SearchSourceResult>> {
  const results: SearchSourceResult[] = [];

  // 1. Tavily 搜索（已集成）
  try {
    const query = buildSearchQuery(config.authorName, config.language);
    const tavilyResults = await searchWeb(query, config.maxResultsPerSource ?? 10);
    results.push(...tavilyResults.map((r) => ({
      source: "tavily" as const,
      title: r.title,
      url: r.url,
      snippet: r.snippet,
      relevance: computeRelevance(r, config.authorName),
    })));
  } catch (e) {
    // Tavily 不可用时静默降级
    console.warn("[author-search] Tavily unavailable:", e instanceof Error ? e.message : e);
  }

  // 2. 未来可扩展：Bing Search API / Google Custom Search
  // if (process.env.BING_API_KEY) { ... }

  // 3. URL 去重：同一 URL 只保留相关度最高的结果
  const urlMap = new Map<string, SearchSourceResult>();
  for (const r of results) {
    const existing = urlMap.get(r.url);
    if (!existing || existing.relevance < r.relevance) {
      urlMap.set(r.url, r);
    }
  }

  return [...urlMap.values()]
    .sort((a, b) => b.relevance - a.relevance)
    .slice(0, 20);
}

/**
 * 根据语言构建更精确的搜索查询。
 */
function buildSearchQuery(authorName: string, language: "zh" | "en"): string {
  const queries: Record<string, string> = {
    zh: `${authorName} 小说 原文 节选 经典段落`,
    en: `${authorName} novel excerpt famous passage prose`,
  };
  return queries[language] ?? queries.zh;
}
```

### 3.2 文风分析维度扩展（需求 2 + 需求 6）

#### 3.2.1 新增维度总览

| 维度 | 当前 | 扩展后 | 实现方式 |
|------|:----:|:------:|---------|
| **句式分布** | ❌ | ✅ 陈述句/疑问句/感叹句/祈使句占比 | 纯函数，基于标点统计 |
| **段落节奏** | ❌ | ✅ 段落长度分布直方图、短段率(<35字)、长段率(>200字) | 纯函数 |
| **修辞细分** | ⚠️ 仅有 `rhetoricDensity` | ✅ 比喻率/排比率/拟人率/夸张率/反问率 | **复用 `detectDuplicateRhetoric()`** |
| **词汇等级** | ❌ | ✅ 常用词占比/文学词占比/方言词占比（基于词频表） | 纯函数 |
| **叙事视角** | ❌ | ✅ 第一人称/第三人称/全知视角占比 | 基于代词统计 |
| **时间表达** | ❌ | ✅ 过去时/现在时/将来时表达密度 | 基于时间词统计 |
| **感官密度** | ✅ 总值 | ✅ 各感官独立 + 示例选段 | 复用 `SensoryBreakdown` |
| **对话特征** | ✅ 占比 | ✅ 平均对话长度/对话频次/对话标签分布 | 基于对话行统计 |

> **词汇等级的实际障碍：** 中文缺乏标准公开的"文学词频表"和"常用词表"。建议采用以下替代方案：
> - **常用词占比**：使用现代汉语常用字表（3500 字）作为基准，统计文本中常用字占比
> - **文学词占比**：使用逆向指标——生僻字（不在 3500 常用字表中）占比
> - **方言词占比**：维护小型方言词词典（如"啥/咋/俺/咱/晓得"等），或标记为 P3 Future

#### 3.2.3 实现策略

所有新增维度为**纯函数**，放在 `packages/core/src/utils/style-dimensions.ts`：

```typescript
// packages/core/src/utils/style-dimensions.ts

export function computeSentenceTypeDistribution(text: string): SentenceTypeDistribution;
export function computeParagraphRhythm(text: string): ParagraphRhythm;
export function computeRhetoricBreakdown(text: string, language: "zh" | "en"): RhetoricBreakdown;
export function computeDialogueFeatures(text: string, language: "zh" | "en"): DialogueFeatures;
export function computeExpandedFingerprint(text: string, language: "zh" | "en"): ExpandedFingerprint;
```

复用关系：
- `computeRhetoricBreakdown` 内部调用 `detectDuplicateRhetoric()`（复用 §3.2.1）
- `computeDialogueFeatures` 分析对话行的长度和标签词分布
- `computeParagraphRhythm` 纯长度统计
- `computeSentenceTypeDistribution` 纯标点统计

#### 3.2.4 `mergeStyleProfiles` 同步扩展清单

| **动作密度** | 找出动作描写最密集的段落 | 他猛地站起身，抓起外套，冲出门去。 |

| **排比句式** | 提取排比结构段落 | 他走了，她走了，他们都走了。 |

#### 3.3.3 示例选段的具体提取算法

以**比喻手法**为例，具体提取步骤如下：

```typescript
function extractMetaphorSamples(
  text: string,
  maxSamples = 2,
): Array<{ text: string; lineNumber: number }> {
  const result = detectDuplicateRhetoric(text, "zh");
  const metaphorFinding = result.findings.find((f) => f.category === "metaphor");
  if (!metaphorFinding) return [];

  return metaphorFinding.examples.slice(0, maxSamples).map((ex) => ({
    text: ex.text,
    lineNumber: ex.lineNumber,
  }));
}
```

以**动作密度**为例：

```typescript
function extractActionSamples(
  text: string,
  maxSamples = 2,
): Array<{ text: string; lineNumber: number }> {
  const sentences = text.split(/[。！？\n]/).filter((s) => s.trim().length > 0);
  const actionScores = sentences.map((s) => ({
    sentence: s,
    score: ACTION_VERBS.reduce((sum, verb) => sum + (s.includes(verb) ? 1 : 0), 0),
  }));

  return actionScores
    .filter((a) => a.score >= 3) // 至少 3 个动作词
    .sort((a, b) => b.score - a.score)
    .slice(0, maxSamples)
    .map((a, i) => ({
      text: a.sentence.slice(0, 120) + (a.sentence.length > 120 ? "…" : ""),
      lineNumber: estimateLineNumber(text, a.sentence),
    }));
}
```

> **通用提取原则：**
> 1. 选段长度控制在 60-120 字，避免过长或过短
> 2. 优先选择包含多个特征词的段落（如动作密度选段应包含 ≥3 个动作动词）
> 3. 避免相邻选段重叠（若两个选段在同一段落内，只保留特征更强的一个）
> 4. 每个维度最多返回 2 段示例，避免信息过载

// packages/studio/src/api/author-sample-writer.ts

export interface SampleMeta {
  readonly authorId: string;
  readonly authorName: string;
  readonly sourceUrl: string;
  readonly fetchedAt: string;
  readonly content: string;
  readonly charCount: number;
}

/**
 * 将抓取到的作家原文样本写入本地 MD 文件。
 * 返回 { filePath, indexUpdated }.
 */
export async function writeAuthorSample(
  root: string,
  meta: SampleMeta,
): Promise<{ filePath: string }> {
  const sampleDir = join(root, "style-library", "authors", meta.authorId, "samples");
  await mkdir(sampleDir, { recursive: true });

  const dateStr = meta.fetchedAt.slice(0, 10).replace(/-/g, "");
  const fileName = `${dateStr}_${sanitizeFileName(extractTitle(meta.sourceUrl))}.md`;
  const filePath = join(sampleDir, fileName);

  const content = [
    `# 样本：${extractTitle(meta.sourceUrl)}`,
    "",
    `**来源：** ${meta.sourceUrl}`,
    `**抓取时间：** ${meta.fetchedAt}`,
    `**字数：** ${meta.charCount.toLocaleString()}`,
    `**文件类型：** 互联网抓取`,
    "",
    "---",
    "",
    meta.content,
    "",
    "---",
    "",
    "*本文档由 InkOS Studio 自动生成，原文版权归原作者所有。*",
    "",
  ].join("\n");

  await writeFile(filePath, content, "utf-8");

  // 更新 sources.json 元数据索引
  await updateSampleSourcesIndex(sampleDir, {
    fileName,
    sourceUrl: meta.sourceUrl,
    fetchedAt: meta.fetchedAt,
    charCount: meta.charCount,
  });

  // 更新 index.md 汇总卡片
  await updateAuthorSampleIndex(sampleDir, meta.authorName, meta.authorId);

  return { filePath };
}
