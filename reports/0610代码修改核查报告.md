### 2.2 BookCreate / BookDetail 扩展（5 个新字段）

#### 新增字段

| 字段 | 类型 | 位置 |
|------|------|------|
| volumeCount | number | BookCreate + BookDetail |
| currentVolume | number | BookCreate + BookDetail |
| keywords | string[] | BookCreate + BookDetail |
| targetAudience | string | BookCreate + BookDetail |
| serializationStatus | enum | BookCreate + BookDetail |

#### 发现的问题

**问题 1：类型逃逸（BookDetail.tsx，5 处）**

```tsx
t("book.keywords" as any)
t("book.targetAudience" as any)
t("common.notSet" as any)
t("book.statusDraft" as any)
t("book.serializing" as any)
t("book.hiatus" as any)
```

- 影响：TypeScript 类型安全丢失
- 建议：在 use-i18n.ts 类型定义中补充这些键，移除 as any
- 严重性：P3

**问题 2：序列化状态翻译键可能不一致**

BookCreate.tsx 定义了 serializationStatusDraft 等键，但 BookDetail.tsx 使用了 statusDraft、serializing、hiatus 等不同键名。需确认两组键是否都已定义。

### 2.3 ChapterReader 章节审计功能

#### 新增功能

- handleAudit()：调用审计端点
- paragraphIssueCount()：将审计问题与段落内容匹配

#### 发现的问题

**问题 3：paragraphIssueCount 性能问题**

- 复杂度 O(n x m)，n=段落数，m=问题数
- stopWords 列表硬编码了 40+ 个中文常用词（"没有"、"可以"、"应该"等），这些词可能是实际内容的一部分
- 建议：提取 stopWords 为常量，考虑更精确的匹配算法
- 严重性：P2

**问题 4：审计按钮 disabled 未绑定**

handleAudit 有 setAuditing(true) 保护，但 UI 按钮未显示 disabled={auditing}
严重性：P3

#### 发现的问题

**问题 5：convertToRhetoricFindings 中 Math.random() 用于生成 ID**

```tsx
id: f.findingId ?? `rhetoric-${f.code}-${Math.random().toString(36).slice(2, 8)}`,
```

- 影响：每次渲染生成新随机 ID，React key 不稳定，可能引发不必要的重渲染
- 建议：使用稳定哈希（如 f.code + f.count + f.samples[0]?.slice(0, 20)）
- 严重性：P2

**问题 6：ReadabilityDashboard 触发条件**

可读性评分仅在 fileText（文件预处理 Tab）有内容时计算，text Tab 的分析结果不会触发。
严重性：P2
