#### A-4 最小代码量实践执行方案

**实施原则**

1. 只修复 URL 与现有 `HashRoute` 状态之间的转换，不调整页面结构。
2. 不修改 `App.tsx`、`ChapterReader`、`BookWorkspace`、后端 API 或 Core。

4. 章节号只接受大于 0 的十进制整数，非法 URL 继续沿用当前策略回退到 Dashboard。

**步骤 1：增加动态路由解析**

在 `parseHash()` 的 service 动态路由之后、book 动态路由之前加入：

```ts
const chapterMatch = path.match(/^chapter\/([^/]+)\/([1-9]\d*)$/);
if (chapterMatch) {
  return {
    page: "chapter",
    bookId: decodeURIComponent(chapterMatch[1]),
    chapterNumber: Number(chapterMatch[2]),
  };
}

const truthMatch = path.match(/^truth\/([^/]+)$/);
if (truthMatch) {
  return {
    page: "truth",
    bookId: decodeURIComponent(truthMatch[1]),
  };
}
```

这里不读取书籍、不检查章节是否存在。路由层只负责解析参数，数据存在性继续由当前页面和 API 处理，避免把业务逻辑引入导航层。

**步骤 2：增加状态到 hash 的序列化**

在 `routeToHash()` 的 `switch` 中增加：

```ts
case "chapter":
  return `#/chapter/${encodeURIComponent(route.bookId)}/${route.chapterNumber}`;
case "truth":
  return `#/truth/${encodeURIComponent(route.bookId)}`;
```

使用现有 `encodeURIComponent` 规则，确保中文和空格书籍 ID 能稳定刷新恢复。

**步骤 3：注册为持久化 hash 页面**

只在现有集合中增加两个值：

```ts
const HASH_PAGES = new Set([
  // existing pages
  "chapter",
  "truth",
]);
```

这一步保证 `nav.toChapter()` 调用 `setRoute()` 时同步写入 URL。当前 `setRouteState(newRoute)` 的即时页面切换逻辑保持不变。

**步骤 4：增加最小回归测试**

至少覆盖以下 8 项：

```ts
expect(parseHash("#/chapter/demo/3"))
  .toEqual({ page: "chapter", bookId: "demo", chapterNumber: 3 });

expect(parseHash("#/chapter/%E4%B9%9D%E9%BE%99/12"))
  .toEqual({ page: "chapter", bookId: "九龙", chapterNumber: 12 });

expect(parseHash("#/truth/demo"))
  .toEqual({ page: "truth", bookId: "demo" });

expect(parseHash("#/chapter/demo/0"))
  .toEqual({ page: "dashboard" });

expect(parseHash("#/chapter/demo/abc"))
  .toEqual({ page: "dashboard" });

expect(routeToHash({ page: "chapter", bookId: "demo", chapterNumber: 3 }))
  .toBe("#/chapter/demo/3");

expect(routeToHash({ page: "truth", bookId: "demo" }))
  .toBe("#/truth/demo");

expect(
  parseHash(routeToHash({ page: "chapter", bookId: "九龙", chapterNumber: 1 })),
).toEqual({ page: "chapter", bookId: "九龙", chapterNumber: 1 });
```

不需要为 `App.tsx` 新增分支测试，因为 chapter 和 truth 的渲染分支已经存在；本次缺陷发生在 URL 转换边界。
