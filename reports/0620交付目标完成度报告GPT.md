## 六、当前最高优先级问题

### P0-交付-1：lint 硬失败

位置：`packages/studio/src/pages/style-manager/useStyleManagerState.ts:247,280`

问题：使用了 `// eslint-disable-next-line react-hooks/exhaustive-deps`，但项目未启用该 eslint rule，导致 disable 注释本身成为 error。

影响：`pnpm -r lint` 不通过，CI / 发布门禁不闭合。

建议：先不要扩大 StyleManager 重构，优先修复 lint hard error。可选择移除无效 disable、改为项目已有 rule、或正式接入 react-hooks eslint 插件，但需要和全项目 lint 策略一致。

### P0-交付-2：Studio agent API 错误语义回归

位置：`packages/studio/src/api/routes/agent.ts` 与 `packages/studio/src/api/server.test.ts`

问题：agent 返回空文本后探测 upstream，fallback 抛错时当前返回 500 / `AGENT_ERROR`，测试期望 502 / 空响应上游错误语义。

影响：API 契约不稳定，前端和调用方无法准确处理模型空响应、额度耗尽或上游异常。

建议：恢复原 server 行为语义，单独捕获 upstream probe 异常并按预期包装，同时补足拆分后 agent route 的回归测试。

### P0-交付-3：CLI 发布打包测试失败

位置：`packages/cli/src/__tests__/publish-package.test.ts`

问题：`npm.cmd pack --pack-destination ...` 在 Studio package prepack 链路退出 2。

影响：无法证明发布包包含 Studio runtime entry 与 built frontend，发布链路不可交付。

建议：单独复现 `npm pack`，展开 npm log，核查 `packages/studio/package.json`、`scripts/prepare-package-for-publish.mjs`、workspace dependency 替换、产物复制和测试临时目录隔离。
