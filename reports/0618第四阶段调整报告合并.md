## 二、三源共识：双实现漂移（🔴 P0 级架构债）

### 2.1 问题本质（DS 主导分析）

Hono 框架使用 **"最后注册胜出"** 的路由匹配策略。当前代码流程：

```
createStudioServer() {
  // 1️⃣ 先注册路由模块
  registerSessionsRoutes → GET/POST/PUT/DELETE /api/v1/sessions/*  ← 被覆盖

  // 2️⃣ 后注册内联路由（server.ts 中 app.get/post/...）
  app.get("/api/v1/sessions", ...)      ← 覆盖了 routes/sessions.ts 的实现
  app.post("/api/v1/agent", ...)        ← 覆盖
}
```

## 三、KM 贡献：第四阶段关键工程修复

### 3.1 CLI 集成测试并发构建竞态修复

**现象**: `pnpm -r test` 并行模式下 CLI 偶发 `ERR_MODULE_NOT_FOUND`

**根因**:
1. `packages/cli/node_modules/@actalk/inkos-core` workspace symlink 丢失
2. `core/package.json` 的 `build` 脚本先 `rm -rf dist` 再 `tsc`，并发时其他包读到不完整的 dist

**修复**（KM 实机执行，尚未 commit）:
- 删除 `cli/node_modules` 后重新 `pnpm install`，恢复 workspace symlinks
- 修改 `packages/core/package.json` 的 `build` 脚本：移除 `fs.rmSync('dist', ...)`，改为直接 `tsc` 覆盖编译

```json
// before
"build": "node -e \"...fs.rmSync('dist',{recursive:true,force:true})\" && tsc"
// after
"build": "node -e \"...fs.rmSync('tsconfig.tsbuildinfo',{force:true})\" && tsc"
```

**验证**: `pnpm -r test` 连续通过，CLI 集成测试 44 项全部通过。

### 3.2 真实 LLM E2E 测试默认跳过开关

**现象**: `.env` 中存在 API Key 时 E2E 自动运行，因上游模型行为不可控而失败/超时，阻塞常规门禁

**修复**（KM 实机执行，尚未 commit）:
- 增加 `INKOS_RUN_REAL_LLM_E2E=1` 显式开关，默认跳过
- 手动触发方式保留：`INKOS_RUN_REAL_LLM_E2E=1 pnpm --filter @actalk/inkos-core test -- src/__tests__/e2e-real-llm.test.ts`

> 此修复与 GPT 报告中 P0-1 建议完全一致——三源达成共识。
