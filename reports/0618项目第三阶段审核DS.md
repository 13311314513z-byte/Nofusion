## 五、发展建议

### 5.1 立即行动（本周）

1. **真实 LLM E2E 回归** — 最高优先级。使用 DeepSeek V4 Flash API key 跑通完整流程：建书→写章→审计→修订→导出。验证 4-5 个 genre 各 1 章。
2. **Studio 前端交互测试** — 手动走查 10 个核心页面，确认无 render crash。
3. **README 更新** — 补充 v1.5.0 功能清单、快速开始指南。

### 5.2 短期（2 周内）

4. **R17 监控接入** — 最小化接入 `pino` + 结构化日志，替代 `console.warn`。Sentry 可选。
5. **测试稳定性** — pipeline-runner.test.ts 抑制预期 stderr；修复偶发超时测试。
6. **CI 验证** — 触发一次完整的 GitHub Actions 运行，确认 Windows smoke job 通过。

### 5.3 中期（1 个月）

7. **性能优化** — 使用 `pnpm bench` 基线数据，设定回归阈值。重点优化 `analyzeStyle` 和 `countChapterLength` 热点路径。
8. **API 版本化** — 当前所有端点为 `/api/v1/`，已验证稳定。建议冻结 v1，新功能走 `/api/v2/`。
9. **Storybook / 组件文档** — 为 ChapterMetadataEditor 等 5+ 个独立组件建立文档。

### 5.4 不建议做的事

- ❌ **runner.ts 完整拆分**：高内聚类，拆分 ROI 为负。已通过 Phase 1 + audit-helpers 达到可维护性目标。
- ❌ **provider.ts 进一步拆分**：43 个供应商适配器已独立。剩余代码为工厂函数 + 传输层，耦合合理。
- ❌ **微服务化**：当前单进程 Hono + Vite 架构满足 1000 章节/天以下的吞吐量需求。过早拆分增加运维复杂度。
