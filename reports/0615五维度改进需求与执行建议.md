## 二、各维度改进需求

### 2.1 character（角色可信度）— 最高优先级

| 现状 | 目标 |
|------|------|
| B 胜率 37.5%，明显落后 | 提升至 ≥ 50%（无显著差距） |
| 角色对话风格不一致 | 同一角色的口气、用词、情绪稳定 |
| 角色行为动机模糊 | 每次出场的行动有可追溯的动机链 |

#### 根因分析

```
1. Writer prompt 未充分注入角色声音约束
2. VoiceProfileAnalyzer 已实现但未被管线消费
3. Auditor 缺少"角色声音偏移"检测维度
4. Writer 生成时未参考角色卡的 arcStage/doNotWrite 字段
```

### 2.2 emotion（情感推进）— 中性

| 现状 | 目标 |
|------|------|
| B 胜率 52.9%，接近中立 | 保持当前水平，无需特别改进 |
| 情感弧线基本自然 | 重点确保关键章节的情感转折不被平淡化 |

#### 保持策略

- 保持当前 `emotional_arcs.md` 的追踪机制
- 在 Writer prompt 中保留情感密度要求
- 不需要额外投入

### 2.3 clarity（叙事清晰度）— 良好

| 现状 | 目标 |
|------|------|

### 2.4 expectation（读者期待）— 良好

### 2.5 engagement（读者参与感）— 强

## 三、改进执行路径

### 3.1 角色声音闭环（建议 2 周）

```
Phase 1：VoiceProfile → Writer 注入（3 天）
  Step 1: VoiceProfileAnalyzer 输出进入 Writer prompt
          → 在 writer.ts 的 assembleWriterContext 中读取 voice_profiles/
          → 将精简规则注入 system prompt
  验收：Writer 生成时参考角色声音规则

Phase 2：Auditor 新增角色声音偏移检测（3 天）
  Step 2: continuity.ts 新增 dim-34 "voice-drift"
          → 检测角色对话偏离已建立的声音画像
          → 严重偏离 → warning
  验收：Audit 报告展示 voice-drift 维度

Phase 3：Studio 前端角色卡编辑器增强（2 天）
  Step 3: BookDetail.tsx 角色卡编辑器增加"声音画像分析"按钮
          → 调用 POST /voice-profiles/analyze
          → 在角色卡面板显示分析结果
  验收：前端可触发声音分析并看到结果

Phase 4：场景模板 → 角色约束联动（2 天）
  Step 4: scene_template.ts 增加角色行为约束字段
          → Writer 在对应场景模板时自动加载约束
  验收：场景模板中的角色约束在 Writer prompt 中生效
```

### 3.2 低温度策略固化

| 改进项 | 说明 | 优先级 |
|--------|------|:------:|
| 默认写作温度从 0.7 → 0.5 | low-temp 在 3/5 维度显著更优 | P1 |
| 保留温度覆盖入口 | CLI `--temperature` 仍可覆盖 | P1 |
| 文档化温度策略 | 在 README 中添加温度说明 | P3 |

### 3.3 审核流程改进

| 改进项 | 说明 |
|--------|------|
| 增加第 2 个 Reader | 已生成 `calibration-results-ensemble.csv`，含 `reader-critic` |
| Ensemble 一致率跟踪 | 每周运行 `stage5-ensemble-eval-sync.mjs` 跟踪变化 |
| 金标数据集 | 使用 `human-labels-template.csv` 积累人工标注 |
