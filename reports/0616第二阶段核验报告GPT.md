根级失败关键错误：

```text
Error: ENOTEMPTY, Directory not empty:
C:\Users\white\Downloads\Nofusion-main\packages\core\dist
```

根因判断：`packages/core/package.json` 的 build 脚本直接执行 `fs.rmSync('dist', { recursive: true, force: true })`，在 Windows 上遇到目录占用或并发读写时会失败。根脚本 `pnpm test` 会先 `pnpm build` 再 `pnpm -r --workspace-concurrency=1 test`，当前仍不能稳定作为发布门禁。

## 三、对照合并报告逐项复核

### 1. Typecheck

合并报告中的 StyleImportTab 类型错误已过期。当前 `pnpm typecheck` 通过，旧 P0 关闭。

仍需注意：StyleManager 拆分主要是 UI 组件拆分，状态和 orchestration 仍集中在父组件，属于可维护性债，不再是类型门禁。

### 2. 场景模板 M5

合并报告判定“API 与 Planner 仅 id/name 共享，场景模板为死数据”。当前代码已变化：

- `SceneTemplateSchema` 包含 `type/location/atmosphere/props/routines/defaultCharacters/notes/tags` 等字段。
- `SceneTemplateRecord` 已镜像主要字段。
- `buildSceneTemplatesBlock()` 已输出 `notes`、`location`、`atmosphere`、`routines`、`props`、`defaultCharacters`。

当前判定：不是死数据，主链路已基本接线。  
剩余缺口：需要一条 API 写入模板 -> Planner prompt 实际包含模板字段 -> 生成计划受影响的集成测试。

### 3. CLI 集成测试

合并报告的 CLI P0 已被当前代码修复。当前 `cli-integration.test.ts` 不再通过 `pnpm.cmd` 子进程启动，而是使用 `process.execPath` 直接执行构建后的 CLI 入口，测试结果全绿。

当前判定：CLI 包级可靠性已恢复。  
剩余缺口：根级 `pnpm test` 仍红，不能把 CLI P0 完全等同于发布闭环完成。

### 4. M10 state_changelog

合并报告认为无 Studio 读取端点。当前已有：

```text
GET /api/v1/books/:id/state-changelog
```

该端点读取 `story/state/state_changelog.jsonl` 并返回 `entries`。所以“无端点”结论已过期。

仍需保留 P1 风险：

- `state-logger.ts` 中 `logStateDelta()` 直接 `appendFile()`，没有在函数内 `mkdir(bookDir/story/state)`。
- 当前 evidence test 主要验证 import 与路径约定，不是实际写入 JSONL 并通过 Studio 端点读回。
- 需要真实写章流程后检查 `state_changelog.jsonl` 产物。

### 5. M3 Plan Alternatives

合并报告称 `write-preview` 硬编码 `planAlternatives: []`。当前代码已改为从 `story/runtime/chapter-XXXX.plan.md` 解析 `## Plan Alternatives`，并提供：

```text
GET /api/v1/books/:id/write-preview
GET /api/v1/books/:id/plan-alternatives
```

当前判定：不是硬编码空数组。  
剩余缺口：只证明“可读已生成 alternatives”，尚未证明前端选择某个 alternative 后能进入 compose/write 的真实控制链。

### 6. G8 genreTags/contentWarnings

合并报告指出空数组无法清空字段。当前 `cleanStringArray()` 对数组输入始终返回数组，空数组语义已修复为“清空”。

当前判定：代码层 P1 已修复。  
剩余缺口：建议补一个 API 测试覆盖 `genreTags: []` 和 `contentWarnings: []` 的持久化。

### 7. CI/CD

`.github/workflows/ci.yml` 存在，合并报告中“CI 文件存在”是正确的。  
但本次未验证 GitHub Actions 真实运行结果，且本地根级 `pnpm test` 仍失败，因此 CI 即使存在也不能视为发布门禁已稳定。
