## 人力测试执行规则

每个按钮和交互控件至少执行一次正常路径；涉及删除、覆盖、批量更新、写章、审计、AI 改写等操作时，再执行一次取消或失败路径。

| 检查维度 | 必查内容 | 判定标准 |
|---|---|---|
| 可发现性 | 按钮名称、图标、悬停提示、入口层级 | 用户能判断用途；纯图标按钮有 `title` 或可访问名称 |
| 点击反馈 | hover、active、loading、disabled | 点击后 300ms 内出现视觉反馈；请求中不可重复提交 |
| 请求反馈 | Network 状态、成功提示、错误提示 | 成功后界面同步；失败时保留上下文并可重试 |
| 取消与回滚 | 弹窗取消、编辑取消、危险操作确认 | 取消不写入；失败不留下半完成状态 |
| 数据一致性 | 列表、详情、统计、正文数据 | 同一实体在不同页面的 ID、数量和状态一致 |
| 持久化 | F5、离开后返回、重启前端 | 已保存内容保留；未保存草稿不伪装成已保存 |
| 路由恢复 | 地址栏 hash、浏览器前进/后退、F5 | URL 与当前页面一致；刷新仍回到相同实体/分区 |
| 边界输入 | 空值、超长、重复、特殊字符、无效文件 | 有校验，不白屏，不产生不可恢复数据 |
| 布局与可访问性 | 1280/1440 宽度、窄屏、键盘 Tab、焦点 | 无遮挡/横向溢出；弹窗焦点和关闭行为明确 |
| 国际化与主题 | 中文、英文、深色、浅色 | 文案不混用；对比度足够；切换后状态不丢失 |

**单项记录格式**：

```text
用例编号：
结果：通过 / 失败 / 阻塞 / 不适用
实际反应：
Console：
Network：
截图或录屏：
缺陷等级：P0 / P1 / P2 / P3
```

**缺陷等级**：

- `P0`：数据丢失、批量误操作、主流程完全不可用。
- `P1`：核心页面错误、前后端数据不一致、保存/创建/写章/审计失败。
- `P2`：局部功能失效、刷新路由丢失、错误提示或交互状态明显不正确。
- `P3`：文案、对齐、提示、可访问性等不阻塞问题。

## 模块 19：全局交互界面穷举审查

本模块不按业务结果重复计数，而是确保每个可交互 DOM 控件都被触发。测试人员在每页打开 DevTools Elements，按页面从上到下执行所有 `button`、链接、下拉、复选框、滑块、文件选择器、菜单项和弹窗操作。

### 19.1 按钮状态四态测试

每个业务按钮记录以下四种状态；不适用时写明原因。

| 状态 | 操作 | 通过标准 |
|---|---|---|
| 初始态 | 页面首次加载 | 文案、图标、禁用条件正确 |
| 悬停/焦点态 | 鼠标悬停、键盘 Tab | 有可见反馈和焦点环 |
| 请求态 | 点击后请求未完成 | loading 可见，防重复提交 |
| 完成/失败态 | 模拟 2xx、4xx/5xx、超时 | 成功同步数据；失败保留输入并可重试 |

### 19.2 菜单、下拉和弹窗

| # | 操作 | 通过标准 |
|---|---|---|
| 19.2.1 | 打开所有「⋮」菜单，再点击菜单外、Esc 和菜单项 | 三种关闭路径有效，焦点返回触发按钮 |
| 19.2.2 | 展开所有模型/服务商/书籍/作者/格式下拉 | 当前项高亮，长列表可滚动，空列表有说明 |
| 19.2.3 | 对所有确认框执行取消、遮罩关闭、确认 | 取消无副作用；危险确认只执行一次 |
| 19.2.4 | 对所有抽屉执行关闭按钮、遮罩、Esc | 内容状态符合设计，不残留透明遮罩 |
| 19.2.5 | 弹窗打开时按 Tab 循环 | 焦点不跑到背景危险按钮 |

### 19.3 表单与输入控件

| # | 操作 | 通过标准 |
|---|---|---|
| 19.3.1 | 所有输入框测试空格、中文、英文、引号、换行、超长值 | 页面不崩溃，校验信息靠近字段 |
| 19.3.2 | 所有数字框测试空值、0、负数、小数、超上限 | 按业务范围阻止或规范化 |
| 19.3.3 | 所有复选框/滑块/预设组合逐项切换 | UI 值、提交 payload、刷新值一致 |
| 19.3.4 | 所有文件选择器测试正确类型、错误类型、空文件、大文件、多文件 | 错误可理解，文件可单独移除 |
| 19.3.5 | API Key 字段测试显隐、粘贴、浏览器自动填充、跨服务商切换 | 不误识别为登录密码，不覆盖其他服务商 |

### 19.4 数据一致性和重复请求

| # | 操作 | 通过标准 |
|---|---|---|

### 19.5 布局审查

每页分别在 `1440×900`、`1280×720`、`1024×768` 和约 `390px` 宽度检查：

- 深色/浅色模式下错误红、警告黄、成功绿的对比度是否足够。
- 空状态、加载骨架、错误状态是否占据稳定布局，避免页面跳动。

## 本轮手册审查结论

- 当前手册已覆盖前端可见导航、直接可访问隐藏路由、服务商与封面配置、新建书籍、旧版书籍设置、BookWorkspace 14 个分区、章节阅读器、聊天、导入导出、文风、审计、体裁、雷达、诊断、日志、自动化、主题和国际化。
- 370 项为可直接勾选的基础业务步骤；模块 19 需要按页面实际控件数量继续展开，因此最终执行次数会高于 370。
- 本次属于文档和前端代码静态对照，不代表浏览器实测已通过。所有结果栏必须由人工执行后填写，不得将“组件已接入”直接等同于“功能通过”。
- 第一批优先验证 `UI-R1` Analytics 刷新路由、`UI-R3` 章节计算按钮嵌套、`UI-R7` API Key/密码管理器，以及首页-概览-章节-正文的数据一致性；这些问题最容易造成真实使用阻塞或错误判断。
