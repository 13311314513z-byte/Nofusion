## 第四部分：分优先级实施方案

### Phase A — 发布修复（P0，2 天）

#### A-1 修复 CLI `chatCompletion` 类型错误（GPT 唯一发现）

**文件**：`packages/cli/src/commands/doctor.ts:329`  
**问题**：`signal` 不在 `chatCompletion` 选项类型中  
**方案**：将超时封装为客户端级的 options 参数，或替换为 `AbortSignal.timeout` 原生支持  
**验收**：根级 `pnpm typecheck` 通过

#### A-2 修复文风 Step 4 崩溃（GPT 唯一发现）

**文件**：`packages/studio/src/pages/StyleManager.tsx`  
**问题**：`/style/rhetoric/rewrite` 返回 `{ prompt }`，前端读取 `{ findings }` → `undefined.filter`  
**方案**：
1. 新增 `/style/rhetoric/detect` 端点固定返回 `{ findings }`
2. `/style/rhetoric/rewrite` 仅接受 findings 返回 `{ prompt }`
3. 前端对数组响应做 `Array.isArray` 防御
4. 增加 Error Boundary 避免单面板崩溃清空页面

#### A-3 修复 Studio 构建产物（三份报告一致）

**文件**：`packages/studio/package.json`  
**问题**：删除 `dist/api/tsconfig.tsbuildinfo` 错误路径，增量缓存跳过服务端输出  
**方案**：
1. 删除 `packages/studio/tsconfig.server.tsbuildinfo`（正确路径）
2. 去掉 `2>nul &` 跨平台兼容性隐患
3. 若仍不可靠，将服务端输出到独立目录（如 `dist-server/`）再拷贝
**验收**：`dist/index.html` + `dist/api/index.js` + `dist/api/index.d.ts` 三者稳定共存

#### A-4 修复章节 hash 路由（GPT 唯一发现）

**文件**：`packages/studio/src/pages/hash-route.tsx`  
**问题**：`chapter` 和 `truth` 路由有类型声明但 `parseHash`/`routeToHash`/`HASH_PAGES` 处理不完整  
**方案**：补全路由序列化/反序列化，允许 `#/chapter/:bookId/:num` 和 `#/truth/:bookId` 直接访问  
**验收**：刷新后章节页和真相页内容正确恢复

#### A-5 修复文风删除/合并行号 Bug（GPT 唯一发现）

**文件**：`packages/studio/src/pages/StyleManager.tsx`  
**问题**：`lineNumber` 为 1 基，前端直接 `lines[lineNumber]` 数组下标（0 基）  
**方案**：改为 `lines[lineNumber - 1]`，段落跨多行时改用稳定字符偏移区间  
**验收**：删除和合并操作实际影响正确行

#### A-5 修复文风删除/合并行号 Bug

**问题定位**  
`DuplicateParagraphGroup` 和 `SimilarParagraphGroup` 的 `lineNumber` 为 1 基（文件行号从 1 开始计数）。前端 `onDelete`/`onMerge` 直接用 `lines[lineNumber]`（数组下标从 0 开始），导致操作偏移一行。

**实施路径**

```typescript
// StyleManager.tsx — onDelete
onDelete={(ids) => {
  const lines = text.split("\n");
  // ids 中的 lineNumber 是 1 基，转为 0 基数组下标
  const indicesToRemove = ids
    .map((id) => id - 1)          // 1基→0基
    .filter((i) => i >= 0 && i < lines.length)
    .sort((a, b) => b - a);       // 从后往前删除
  
  for (const idx of indicesToRemove) {
    lines[idx] = "";
  }
  setText(lines.filter((l) => l !== "").join("\n"));
}}
```

```typescript
// StyleManager.tsx — onMerge（SimilarParagraphGroup 使用 paragraphs[0].lineNumber）
onMerge={(group, mergedText) => {
  const lines = text.split("\n");
  if (group.paragraphs && group.paragraphs.length > 0) {
    const firstLineIndex = group.paragraphs[0].lineNumber - 1; // 1基→0基
    if (firstLineIndex >= 0 && firstLineIndex < lines.length) {
      lines[firstLineIndex] = mergedText;
    }
    const restIndices = group.paragraphs
      .slice(1)
      .map((p) => p.lineNumber - 1)   // 1基→0基
      .filter((i) => i >= 0 && i < lines.length)
      .sort((a, b) => b - a);
    
    for (const idx of restIndices) {
      lines[idx] = "";
    }
  }
  setText(lines.filter((l) => l !== "").join("\n"));
}}
```

**验收标准**  
- 文本第 1 行（lineNumber=1）可被正确删除（而非删除第 2 行）
- 段落跨多行时，删除操作不影响相邻段落
- 合并相似段落时，合并内容替换正确段落

```typescript
  let hash = 0;
  for (let i = 0; i < s.length; i++) {
    hash = ((hash << 5) - hash) + s.charCodeAt(i);
    hash |= 0;
  }
  return Math.abs(hash).toString(36);
}
```

```typescript
// packages/studio/src/api/server.ts — PUT /books/:id/chapters/:num handler

import { mkdir, writeFile, readFile } from "node:fs/promises";
import { join, dirname } from "node:path";

// 在现有 PUT handler 中，于 writeFile 之前插入备份逻辑
app.put("/api/v1/books/:id/chapters/:num", async (c) => {
  const id = c.req.param("id");
  const num = parseInt(c.req.param("num"), 10);
  const body = await c.req.json<{ content: string }>();
  
  try {
    // 1. 读取旧内容作为 backup（如果存在）
    const chapterPath = await findChapterFile(root, id, num);
    let oldContent = "";
    if (chapterPath) {
      oldContent = await readFile(chapterPath, "utf-8");
    }
    
    // 2. 写入新内容
    if (chapterPath) {
      await writeFile(chapterPath, body.content, "utf-8");
    }
    
    // 3. 如果有旧内容且已变化，保存 backup
    if (oldContent && oldContent !== body.content) {
      // 读取当前 revision 号
      const metaPath = join(root, "books", id, "chapters", "index.json");
      let revision = 1;
      try {
        const index = JSON.parse(await readFile(metaPath, "utf-8")) as { chapters: Array<{ number: number; revisionCount?: number }> };
        const chapterMeta = index.chapters.find((ch) => ch.number === num);
        revision = (chapterMeta?.revisionCount ?? 0) + 1;
        
        // 更新 revisionCount
        if (chapterMeta) {
          chapterMeta.revisionCount = revision;
        }
        await writeFile(metaPath, JSON.stringify(index, null, 2), "utf-8");
      } catch { /* 无元数据时默认 revision=1 */ }
      
      // 写入版本文件
      const versionDir = join(root, "books", id, "story", "chapter-versions", String(num));
      await mkdir(versionDir, { recursive: true });
      await writeFile(join(versionDir, `${revision}.md`), oldContent, "utf-8");
    }
    
    return c.json({ ok: true });
  } catch (e) {
    return c.json({ error: String(e) }, 500);
  }
});
```

```typescript
/**
 * GET /api/v1/books/:id/chapters/:num/versions
 * List all backup versions of a chapter.

app.get("/api/v1/books/:id/chapters/:num/versions", async (c) => {
  const id = c.req.param("id");
  const num = c.req.param("num");
  const versionDir = join(root, "books", id, "story", "chapter-versions", num);
  
  try {
    const files = await readdir(versionDir);
    const versions = files
      .filter((f) => f.endsWith(".md"))
      .map((f) => ({
        revision: parseInt(f.replace(".md", ""), 10),
        file: f,
        createdAt: "",  // 可从文件 stat 获取
      }))
      .filter((v) => !isNaN(v.revision))
      .sort((a, b) => b.revision - a.revision);
    
    // 为每个版本获取时间戳
    for (const v of versions) {
      try {
        const stat = await lstat(join(versionDir, v.file));
        v.createdAt = stat.birthtime.toISOString();
      } catch { /* ignore */ }
    }
    
    return c.json({ versions });
  } catch {
    return c.json({ versions: [] });
  }
});
```
