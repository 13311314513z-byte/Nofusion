## 二、循迹发现的额外问题

### 2.1 BookDetail.tsx — alert() 泛滥（P1）

**发现**：20 处 `alert()` 调用（确认/错误/信息提示全部使用原生弹窗）+ 1 处 `window.confirm()`

| 操作 | 当前方式 | 建议 |
|------|---------|------|
| 删除书籍 | `alert(e.message)` | 统一 Toast 组件 |
| 章节审批/驳回 | `alert("Approve/Reject failed")` | 内联错误 + 禁用态 |
| 导出保存 | `alert(\`Exported: ${path}\`)` | Toast 通知 |
| 角色删除确认 | `window.confirm(...)` | 统一 ConfirmDialog |
| 检索角色/新建角色 | `alert(e.message)` | Toast |

**影响**：原生 `alert()` 阻塞 UI 线程，无样式，无堆叠，用户体验差。`window.confirm` 与项目中已经存在的 `ConfirmDialog` 组件不统一。

### 2.2 BookChaptersSection.tsx — 批量操作防重入缺失（P1）

**发现**：
- 单章审批/驳回按钮 `onClick` 为 async 但无 `disabled` 绑定
- 批量地点/POV/标签操作使用 `setBulkProgress` 做进度追踪，但按钮自身无 `disabled`
- 「计算全部文风分」的取消按钮通过全局 ref 控制而非 React state

### 2.3 StyleManager — 多处 async 操作无保护（P1）

| 操作 | 位置 | 问题 | 建议 |
|------|------|------|------|
| 作家重分析 | line 1592 | async onClick 无 disabled | 加 `reanalyzing` state |
| 调整计划生成 | line 1008+ | async onClick 无 disabled | 加 `generating` state |
| 导入 URL | line 260 | async onClick 无 disabled | 加 `loading` state |
| 删除作者 | line 715 | async onClick 无 disabled | 加 `deleting` state |

## 三、循迹路径与验证方法

以下为本次循迹使用的 grep 模式，供团队复现：

```bash
# 查找 async onClick 按钮（潜在防重入缺失）
grep -rn "onClick={async" packages/studio/src/pages/ --include="*.tsx" | grep -v "node_modules" | wc -l
# 结果：43 处 async onClick

# 查找其中带 disabled 保护的
grep -rn "onClick={async" packages/studio/src/pages/ --include="*.tsx" | grep "disabled" | wc -l
# 结果：约 31 处有 disabled

# 差值：12 处无 disabled 保护

# 查找 alert() 使用
grep -rn "alert(" packages/studio/src/ --include="*.tsx" --include="*.ts" | grep -v "__tests__" | grep -v "node_modules" | wc -l
# 结果：~47 处（含测试文件）

# 查找 window.confirm
grep -rn "window.confirm\|\.confirm(" packages/studio/src/ --include="*.tsx" | grep -v "node_modules" | wc -l
# 结果：1 处
```
