## 4. 阻断缺陷

### P0-01：当前源码前端白屏

**现象**

- `http://localhost:4577` 返回 HTTP 200。
- 页面 `#root` 内容长度为 0，正文为空。
- 浏览器控制台出现 `ReferenceError: process is not defined`。
- 异常来源进入了浏览器侧 `undici.js`。
- 同时出现 `node:http`、`node:buffer`、`node:querystring`、`node:events` 被浏览器外部化的警告。

**根因证据**

- `packages/studio/src/pages/StyleManager.tsx:19` 从 `@actalk/inkos-core` 根入口运行时导入文风函数。
- Core 根入口同时导出了服务器侧模块。
- `packages/core/src/agents/rules-reader.ts` 使用 `node:fs/promises`、`node:path`、`node:url`。
- 浏览器打包因此拉入 Node 专用依赖，最终在运行时访问不存在的 `process`。

**影响**

- 当前源码的首页和全部前端模块不可用。
- 手册要求的逐模块、逐按钮测试无法在当前源码版本完成。
- 开发环境虽然端口启动成功，但不具备可使用性。

**建议**

- 为 Core 划分浏览器安全入口，例如 `@actalk/inkos-core/browser`。
- Studio 只从浏览器入口导入纯函数和类型，禁止经根入口引入文件系统、网络和 CLI 依赖。
- 在 `package.json exports` 中明确 browser/node 子路径。
- 增加 Studio 首屏烟雾测试：加载后断言 `#root` 非空且无 `pageerror`。

### P1-01：Studio 生产构建失败

**错误**

`rules-reader.js` 导入的 `join` 无法从 Vite 的 `__vite-browser-external:node:path` 导出。

构建同时检测到 `node:fs/promises`、`node:path`、`node:url` 进入浏览器依赖图。构建在转换约 6868 个模块后失败。

**影响**

- 无法生成当前源码对应的生产前端。
- 发布包集成测试失败。
- 4579 上可访问的前端只能视为既有构建产物，不能作为当前源码构建通过的证据。

- `type="password"`
- 使用非通用随机化名称
- `autocomplete="off"`

- `type="password"`
- 名称包含服务商 ID

判断：P2，建议统一复用服务详情页的 API Key 输入组件，并增加 `autocomplete="off"` 或更严格的令牌字段策略。

**影响**

### P2-02：章节批量文风计算存在嵌套按钮

`packages/studio/src/pages/book-workspace/BookChaptersSection.tsx:676-695` 的“计算全部文风分数”外层是 `<button>`，计算中又在内部渲染“取消” `<button>`。

这是无效 HTML 交互结构，可能导致点击冒泡、键盘访问和浏览器兼容性问题。建议将外层改为容器，或把取消按钮移到同级。

### P2-03：空书或不完整书籍导出返回通用 500

测试前 5 本书时：

- `test-book-0609` 导出成功，响应约 15148 字节。
- 4 本无章节或不完整书籍返回 `500 {"error":"Export failed"}`。

建议在导出前验证章节数量和必要配置，返回可操作的 4xx 错误，例如“当前书籍没有可导出的章节”，不要统一包装成 500。

### P2-04：国际化存在硬编码残留

英文界面仍可见中文标签：

- `AI痕迹`
- `文风诊断`

静态检查还发现部分错误提示按钮使用硬编码 `Dismiss`。建议所有可见文本统一进入 i18n 字典，并增加中英文页面文本快照检查。

### P3-01：未知体裁 ID 静默回退

`GET /api/v1/genres/not-a-genre` 返回 HTTP 200，并给出通用 `other` 体裁，而不是 404。

这可能掩盖前端传错 ID。若该行为是兼容设计，应在响应中明确 `fallback=true`；否则建议返回 404。
