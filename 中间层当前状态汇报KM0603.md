## 二、P0 级问题（已在本轮审核中修复）

### 2.1 `audit-history.ts` — 审计历史 jsonl

| 问题 | 风险 | 修复方式 |
|---|---|---|
| `auditResult.issues` 假设永远存在，运行时若为 `undefined` 直接 `TypeError: Cannot read properties of undefined (reading 'filter')` | audit 流程 crash | `(auditResult.issues ?? []).filter(...)` |
| 单条 JSONL 损坏导致整文件历史被丢弃（`loadAuditHistory` 用外层 catch 吞掉所有 `JSON.parse` 异常） | 审计历史静默丢失 | 逐行 `try-catch`，跳过损坏行，保留其余 |
| `loadAuditHistory` 使用动态 `import("node:fs/promises")`，与项目静态 import 风格不一致 | 打包/运行时解析开销 | 改为文件顶部静态 import |
| 裸 `catch {}` 把 `EACCES`/`ENOTDIR` 与 `ENOENT` 同等处理 | 权限问题无法排查 | 区分 `ENOENT` 与其他错误，其他错误重新抛出 |

### 2.2 `chapter-goal.ts` — 章节目标卡

| 问题 | 风险 | 修复方式 |
|---|---|---|
| `saveChapterGoals` 直接 `writeFile`，进程崩溃时产生截断文件，下次加载回退到空数组 | 用户数据丢失 | 临时文件 + `rename` 原子写入 |
| `bookDir` 为空字符串时 `join("", "story", FILENAME)` 生成相对路径，可能误写工作目录 | 数据写入错误位置 | 增加 `if (!bookDir) throw` 校验 |
| `loadChapterGoals` 仅校验 `Array.isArray(parsed.goals)`，不校验元素类型 | 类型污染，消费端难追踪 | 增加运行时断言，过滤非法元素 |

### 2.3 `role-card.ts` — 角色卡模板

| 问题 | 风险 | 修复方式 |
|---|---|---|
| `parseFrontmatter` 将任何以 `-` 开头的值误判为列表开始：`arcStage: -1` / `name: -Alice` 的值被完全丢弃 | 数据静默丢失 | 区分 `"- "`（列表项）与 `"-1"`（负数值/字符串） |
| `stringifyFrontmatter` 未对值进行 YAML 安全转义，含 `: # \n` 的字段破坏 frontmatter 结构 | 重新解析时数据损坏 | 增加 `yamlScalar()` 函数，对特殊字符用 JSON.stringify 转义 |
| `saveRoleCard` 先遍历删除 legacy 文件再写入，中间崩溃导致文件完全消失 | 角色卡丢失 | 改为先原子写入目标文件，成功后再清理 legacy |

### 2.4 `chapter-import-planner.ts` — 章节导入预览

| 问题 | 风险 | 修复方式 |
|---|---|---|
| `startNumber` 可传入负数，导致 `targetNumber` 为负 | 与业务语义冲突 | `Math.max(1, Math.floor(...))` |
| 非法 `splitRegex`（如 `"["`）直接透传 `SyntaxError` 给调用方，server 层返回 500 而非 400 | 用户无法感知输入错误 | 内部 `try-catch`，返回带 `invalid-regex` 警告的空结果 |
| `countCjkChars` 正则 `[\u4e00-\u9fa5]` 仅覆盖基本汉字区，生僻字统计偏低 | 字数统计偏差 | 增加 CJK Extension A `[\u3400-\u4dbf]` |

## 三、P0 级问题（未修复，需关注）

### 3.1 `audit-history.ts` — 并发写入风险

- **问题**：`appendAuditHistory` 使用 `fs.appendFile` 非原子追加。同一书籍若并行运行多个 pipeline（极少见但可能），多条 JSONL 可能字节级交错，产生不可解析的混穿行。
- **影响**：审计历史损坏，需人工修复。
- **建议**：引入文件级互斥锁（如项目现有模式中的队列机制），或按 `chapterNumber` 分片写入。
- **优先级**：P1（正常单 pipeline 顺序执行不会触发）

### 3.2 `chapter-goal.ts` / `role-card.ts` — 并发覆盖风险

- **问题**：两者均缺少并发保护。若两个调用同时 `saveChapterGoals` 或 `saveRoleCard`，后写入者覆盖前者。
- **影响**：更新丢失。
- **建议**：引入调用方队列或文件锁。
- **优先级**：P1（单用户操作不会触发，多终端/快速连点可能触发）
